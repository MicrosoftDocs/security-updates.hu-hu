Security Bulletin

A Microsoft számú biztonsági közleménye MS10-085 - Fontos
=========================================================

Szolgáltatásmegtagadást lehetővé tévő SChannel biztonsági rés (2207566)
-----------------------------------------------------------------------

Közzétéve: 2010. október 12. | Frissítve: 2010. október 18.

**Verzió:** 2.0

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Windows biztonságos csatorna (SChannel) biztonsági csomagjának közvetlenül jelentett biztonsági rését szünteti meg. A biztonsági rés szolgáltatásmegtagadást tehet lehetővé, amennyiben az érintett rendszer a biztonságos szoftvercsatornán (SSL) keresztül speciálisan kialakított csomag üzenetet kapott. Alapértelmezés szerint a Windows Vista, a Windows Server 2008, a Windows 7 és a Windows Server 2008 R2 egyetlen támogatott kiadásán sincs konfigurálva az SSL hálózati forgalom fogadása.

Ez a fontos besorolású biztonsági frissítés a Windows Vista, Windows Server 2008, Windows 7 és a Windows Server 2008 R2 összes támogatott kiadását érinti. További információkért lásd az **Érintett és nem érintett szoftverek** alfejezetet.

A biztonsági frissítés a biztonsági rés megszüntetése érdekében módosítja az SChannel protokollkérésekre vonatkozó feldolgozási módját a bejövő SSL-kapcsolatok kezelése során. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft a minél előbbi telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** Nincsenek

TLSv1 szolgáltatásmegtagadási biztonsági rés - CVE-2010-3229  
------------------------------------------------------------
  
<span></span>
Az SChannel protokollkérésekre vonatkozó feldolgozási módja bejövő SSL-kapcsolatok esetén szolgáltatásmegtagadást lehetővé tévő biztonsági rést okoz Windows Server 2008, Windows Vista, Windows Server 2008 R2 és Windows 7 rendszeren. A távoli, ismeretlen támadó speciálisan kialakított hálózati csomagot küldhet az érintett rendszerre, mely az LSASS szolgáltatás válaszadásának leállását és a rendszer újraindulását okozhatja. Csak azok a rendszerek érintettek, melyeken a szolgáltatás vagy az alkalmazás konfigurálva van SSL hálózati forgalom fogadására.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2010-3229](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3229).
  
#### A TLSv1 szolgáltatásmegtagadási biztonsági rés hibáit enyhítő tényezők - CVE-2010-3229
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   A biztonsági rés csak olyan rendszereken használható ki, melyeken egy alkalmazás vagy szolgáltatás konfigurálva van bejövő SSL-kapcsolatok fogadására. A biztonsági rés nem érinti azokat a rendszereket, melyek csak kimenő SSL-kapcsolatokat kezdeményezhetnek.  
-   Alapértelmezés szerint a Windows Vista, a Windows Server 2008, a Windows 7 és a Windows Server 2008 R2 egyetlen támogatott kiadásán sincs konfigurálva az SSL hálózati forgalom fogadása.
  
#### A TLSv1 szolgáltatásmegtagadási biztonsági rés lehetséges megoldásai - CVE-2010-3229
  
A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### A TLSv1 szolgáltatásmegtagadási biztonsági réssel kapcsolatban gyakran ismételt kérdések - CVE-2010-3229
  
**Mire terjed ki a biztonsági rés hatása?**    
Ez a biztonsági rés szolgáltatásmegtagadást okoz. A biztonsági rést kihasználó támadó leállíthatja az LSASS szolgáltatást és újraindíthatja a számítógépet, ezáltal a rendszer sikeres újraindulásáig szolgáltatásmegtagadást okozhat. A támadó a biztonsági rést többször is kihasználhatja.
  
**Mi okozza a biztonsági rést?**    
A biztonsági rést az okozza, hogy az SChannel az SSL-/TLS-kézfogás során, a protokollkérelem feldolgozása előtt nem ellenőriz egy logikai feltételt.
  
**Mi az SChannel biztonsági csomag?**    
A biztonságos csatorna (SChannel) biztonsági csomag olyan biztonságitámogatás-szolgáltató (SSP), mely a Secure Sockets Layer (SSL) és Transport Layer Security (TLS) szabványos internetes hitelesítési protokollokat alkalmazza. Ezek az összetevők a számos általános internetes és hálózati alkalmazást, például a böngészést támogató biztonságos kommunikáció lebonyolítására szolgálnak. Az SChannel annak a biztonsági csomagnak a része, amely hitelesítési szolgáltatást nyújt az ügyfél és a kiszolgáló közti biztonságos kommunikáció megvalósításához.
  
További információért lásd a [Secure Channel](http://msdn.microsoft.com/en-us/library/aa380123(vs.85).aspx) című MSDN cikket.
  
**Mi a TLS és az SSL?**    
A Transport Layer Security (TLS) kézfogás protokollja felelős a biztonságos munkamenetek létrehozásához vagy folytatásához szükséges hitelesítéséért és kulcscseréért. A Secure Sockets Layer (SSL) a Transport Layer Security protokoll elődje. A TLS és az SSL azonos funkciókat lát el, és a nyilvános/titkos kulcs technológiát ötvöző eljárással támogatja a biztonságos hálózati kommunikációt.
  
További információk a [TSL/SSL mikéntjéről szóló](http://technet.microsoft.com/en-us/library/cc783349(ws.10).aspx) TechNet cikkben.
  
**Mi a TLS- vagy SSL-kézfogás?**    
A TLS vagy SSL használatával létrehozott biztonságos munkamenet esetén a kézfogás protokoll kezeli a következőket:
  
-   rejtjelező csomag egyeztetése  
-   a kiszolgáló és (opcionálisan) az ügyfél hitelesítése  
-   munkamenetkulcsok információcseréje
  
További információk a [TSL/SSL mikéntjéről szóló](http://technet.microsoft.com/en-us/library/cc783349(ws.10).aspx) TechNet cikkben.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó a válaszadások megszakítására és újraindításra kényszerítheti a kiszolgálót.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A távoli, ismeretlen támadó speciálisan létrehozott üzenetcsomagot küldhet a rendszeren futó SSL-kompatibilis szolgáltatásnak vagy alkalmazásnak. A biztonsági rés csak akkor használható ki, ha a szolgáltatás vagy az alkalmazás SChannel csatornát használ az SSL-forgalom feldolgozására. SSL-kompatibilis szolgáltatások többek között, de nem kizárólagosan az Internet Information Services (IIS), a Remote Desktop Protocol (RDP) és a Secure Socket Tunneling Protocol (SSTP), továbbá az olyan egyéb bújtatási alkalmazások, mint az SMTP-over-SSL vagy az LDAP-over-SSL. A támadó minden esetben speciálisan kialakított csomagot küld az SSL-kézfogás részeként. A sikeres támadás leállítja az LSASS-szolgáltatás válaszadását és újraindítja a rendszert, amely ideiglenes szolgáltatásmegtagadást okoz. A rendszer újraindulását követően a támadó ismét kihasználhatja a biztonsági rést.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
A biztonsági rés a Microsoft Windows olyan érintett verzióját futtató rendszereken használható ki, melyeken egy alkalmazás vagy szolgáltatás konfigurálva van bejövő SSL-kapcsolatok fogadására. Többek között, de nem kizárólagosan az SSL-kapcsolatokat elfogadó webkiszolgálókként, a bejövő SSTP-kapcsolatokat elfogadó VPN-kiszolgálókként, illetve a Távoli asztal szolgáltatások szerepkörben működő kiszolgálókként konfigurált rendszerek érintettek. Szintén veszélyeztetettek a Microsoft Windows azon verzióit futtató munkaállomások, melyeken a Remote Desktop Protocol (RDP) engedélyezett.
  
Kevésbé veszélyeztetettek azon rendszerek, melyek csak kezdeményezhetnek, de nem fogadhatnak SSL-kapcsolatokat. A kevésbé veszélyeztetett rendszerek közé tartoznak például azon munkaállomások, melyekről a felhasználó a webes böngészéshez, illetve azon laptop számítógépek, melyekről a felhasználó az internetkapcsolathoz VPN-ügyfélként csatlakozik. Fontos tudni, hogy ezen ügyfélrendszereken is futhatnak olyan alkalmazások vagy szolgáltatások, melyek elfogadják a bejövő SSL-kapcsolatokat, ebben az esetben ezek a rendszerek is veszélyeztetettek.
  
**Mi a frissítés feladata?**    
A frissítés a biztonsági rés megszüntetése érdekében módosítja az SChannel protokollkérésekre vonatkozó feldolgozási módját a bejövő SSL-kapcsolatok kezelése során.

#### Microsoft Active Protections Program (MAPP)
  
Az ügyfelek védelmének növelése érdekében a biztonsági szoftverekkel foglalkozó nagyobb gyártók a Microsoft a havi biztonsági közlemények megjelenése előtt ellátja biztonsági résekkel kapcsolatos információkkal. Így a biztonságiszoftver-gyártók a kapott biztonsági résekkel kapcsolatos információt felhasználhatják ügyfeleik hatásosabb védelmére az olyan biztonsági szoftverek vagy eszközök (például víruskeresők, hálózati behatolásérzékelő rendszerek vagy kiszolgálóalapú behatolásvédelmi rendszerek) frissítése által. A biztonságiszoftver-gyártók által kiadott aktív védelem elérhetőségéről a [Microsoft Active Protections Program (MAPP) Partners](http://www.microsoft.com/security/msrc/mapp/partners.mspx) webhelyén található listán szereplő és a programban részt vevő gyártók aktív védelmi webhelyein tájékozódhat.
