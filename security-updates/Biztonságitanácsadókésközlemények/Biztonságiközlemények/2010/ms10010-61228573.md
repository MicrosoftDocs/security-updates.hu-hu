Security Bulletin

A Microsoft számú biztonsági közleménye MS10-010 - Fontos
=========================================================

A Windows Server 2008 Hyper-V biztonsági rése szolgáltatásmegtagadást okozhat (977894)
--------------------------------------------------------------------------------------

Közzétéve: 2010. február 9. | Frissítve: 2010. február 10.

**Verzió:** 1.1

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a közvetlenül jelentett biztonsági rés hibáit oldja meg a Windows Server 2008 Hyper-V és a Windows Server 2008 R2 Hyper-V használatával működő kiszolgálókon. A biztonsági rés szolgáltatásmegtagadást tehet lehetővé, ha a Hyper-V kiszolgálón vendégként futó virtuális gépek valamelyikében hitelesített felhasználó rosszindulatúan kialakított gépi utasítások sorozatát hajtja végre. A támadónak a biztonsági rés kihasználásához érvényes bejelentkezési adatokkal kell rendelkeznie, és képesnek kell lennie helyileg bejelentkezni a virtuális gépbe. Névtelen felhasználók nem tudják távolról kiaknázni ezt a biztonsági rést.

A biztonsági frissítés a Windows 2008 Server és a Windows Server 2008 R2 összes támogatott x64 alapú kiadására nézve fontos besorolásúnak minősül. További tudnivalókért lásd az **Érintett és nem érintett szoftverek** c. részt.

A biztonsági frissítés úgy szünteti meg a biztonsági rést, hogy kijavítja azt a módot, ahogyan a Hyper-V kiszolgáló a vendégként futó virtuális gépein végrehajtott gépi utasítások kódját érvényesként fogadja el. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft a minél előbbi telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** Nincsenek

**Hol találhatók a fájlinformációs adatok?**  
A fájlinformációs adatokat lásd a **Biztonsági frissítés telepítése** rész hivatkozási táblázataiban.

**Felajánlják ezt a frissítést akkor is, ha nálam nincs engedélyezve a Hyper-V?**
**Ez a frissítés manuálisan telepíthető az** érintett platformokra akkor is, ha a Hyper-V szerep nincs engedélyezve. Az Automatikus frissítés során azonban csak akkor kínálják fel, ha a Hyper-V szerep engedélyezve van.

**Kell-e ezt a frissítést a Hyper-V vendég virtuális gépekre is alkalmazni?**
Ezt a frissítést a Hyper-V vendég virtuális gépekre nem kell alkalmazni, csak arra a gépre kell, amelyiken a Hyper-V szerep engedélyezve van. Az Automatikus frissítés során csak a Hyper-V kiszolgálórendszerekre történik felajánlás, a virtuálizált vendéggépeken belül nem.

**Mi a Hyper-V Server, és azt is érinti ez a biztonsági rés?**
A Microsoft Hyper-V Server olyan önálló termék, amelyik a megbízható és optimalizált virtualizálási megoldásával lehetővé teszi, hogy a szervezetek jobban kihasználják a kiszolgálóikat és csökkentsék a költségeiket. Mivel a Hyper-V Server olyan önálló terméknek szánt, amelyik csak a Windows Hypervisor, a Windows Server illesztőmodelljét és virtualizálási összetevőit tartalmazza, kis ráfordítást és minimális terhelést jelent.
A Microsoft Hyper-V Server 2008, a Microsoft Hyper-V Server 2008 Service Pack 2 és a Microsoft Hyper-V Server 2008 R2 érintett ebben a biztonsági résben, és a Windows Server 2008 és a Windows Server 2008 R2 frissítéseit alkalmazni kell a jelen biztonsági rés orvosolásának elősegítése érdekében. Akiknél engedélyezve van az Automatikus frissítés, automatikusan ajánlást kapnak ezen frissítések telepítésére.

A Hyper-V utasításkészletének érvényesítése miatti biztonsági rés - CVE-2010-0026  
---------------------------------------------------------------------------------
  
<span></span>
A szolgáltatásmegtagadásban a Windows Server 2008 és a Windows Server 2008 R2 rendszeren futó Hyper-V érintett. A biztonsági rést az okozza, hogy elégtelen a Hyper-V részéről a gépi utasítások bizonyos sorozatainak az érvényesítése. A biztonsági rést sikeresen kihasználó támadó az érintett Hyper-V rendszert a válaszadások megszakítására kényszerítheti. Ez érintheti az adott rendszeren futó összes virtuális gépet.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2010-0026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0026).
  
#### A Hyper-V utasításkészletének elégtelen érvényesítése miatti biztonsági rés hibáit enyhítő tényezők - CVE-2010-0026
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   A támadónak a biztonsági rés kihasználásához érvényes bejelentkezési adatokkal kell rendelkeznie, és képesnek kell lennie helyileg bejelentkezni a Hyper-V virtuális gépbe. Névtelen felhasználók nem tudják távolról kiaknázni ezt a biztonsági rést.
  
#### A Hyper-V utasításkészletének elégtelen érvényesítése miatti biztonsági rés elkerülése - CVE-2010-0026
  
A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### Gyakran feltett kérdések a Hyper-V utasításkészletének elégtelen érvényesítése miatti biztonsági réssel kapcsolatban - CVE-2010-0026
  
**Mire terjed ki a biztonsági rés hatása?**    
Ez a biztonsági rés szolgáltatásmegtagadást okoz. A biztonsági rést sikeresen kihasználó támadó az érintett Hyper-V kiszolgálót a válaszadás megszakítására és újraindításra kényszerítheti. Fontos megjegyezni, hogy a szolgáltatásmegtagadást okozó biztonsági rések a támadó számára nem teszik lehetővé programkód futtatását vagy jogosultságok illetéktelen megszerzését, de segítségükkel megakadályozható a rendszerben a kérelmek fogadása.
  
**Mi okozza a biztonsági rést?**    
Ezt a biztonsági rést az okozza, hogy a Hyper-V kiszolgáló helytelenül érvényesíti a virtuális vendéggépeken belül végrehajtott bizonyos gépi utasítások kódolását. Az érvényesítés ezen hiányossága miatt ezeknek az utasításoknak a feldolgozása miatt a Hyper-V kiszolgáló alkalmazás válaszmegtagadóvá válhat.
  
**Mi a Hyper-V?**    
A Hyper-V olyan hipervizor alapú technológia, amelyik a Windows Server 2008 és a Windows Server 2008 R2 kulcsfontosságú jellemzője. Ez biztosítja a méretezhető, megbízható és stabilan rendelkezésre álló virtualizálási platformot. A virtualizálási környezet hasznos, mert a felhasználók létrehozhatnak és kezelhetnek virtuális gépeket, amelyek lehetővé teszik, hogy fizikailag egyetlen számítógépen több operációs rendszert futtassanak és az operációs rendszereket egymástól elszigeteljék.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó a felhasználó rendszerének újraindítást igényló lefagyását idézheti elő. Tudnia kell, hogy a biztonsági rés kihasználása a Hyper-V kiszolgálójának, beleértve az adott kiszolgálón lévő minden virtuális vendéggépet is, lefagyását eredményezheti.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A támadónak a Hyper-V kiszolgálón lévő virtuális vendéggépek valamelyikén hitelesített felhasználónak kell lenni, és arra is lehetőségének kell lenni, hogy a rendszeren végrehajthasson tetszőleges kódot. A támadó ekkor elindíthat egy nem biztonságos végrehajtható fájlt a rendszeren, amelyik rosszindulatúen kialakított gépi utasítások sorozatát hívja meg, és ezzel a Hyper-V kiszolgáló lefagyását okozhatja.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
A biztonsági rés különösen azokra a Hyper-V kiszolgálókra veszélyes, amelyek olyan virtuális vendéggépeket működtetnek, amelyeket nem megbízható felhasználók is elérhetnek.
  
**Mi a frissítés feladata?**    
A frissítés úgy szünteti meg a biztonsági rést, hogy kijavítja azt a módot, ahogyan a Hyper-V kiszolgáló a vendégként futó virtuális gépein végrehajtott gépi utasítások kódját érvényesként fogadja el.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**    
Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**    
Nem. A Microsoft nem kapott olyan információt, hogy a biztonsági közlemény eredeti változatának megjelenéséig a biztonsági rést felhasználták volna nyilvános támadásra.
