Security Bulletin

A Microsoft számú biztonsági közleménye MS09-070 - Fontos
=========================================================

Az Active Directory összevonási szolgáltatásaihoz kapcsolódó biztonsági rések távoli programkódfuttatást tehetnek lehetővé (971726)
-----------------------------------------------------------------------------------------------------------------------------------

Közzétéve: 2009. december 8. | Frissítve: 2009. december 9.

**Verzió:** 1.1

### Általános tudnivalók

#### Összefoglalás

Ez a biztonsági frissítés a Microsoft Windows két közvetlenül jelzett biztonsági rését szünteti meg. E biztonsági rések súlyosabbjai távoli kódfuttatást tesznek lehetővé, ha a támadó speciálisan kialakított HTTP kérelmeket küld valamelyik ADFS-képes webkiszolgálóra. A biztonsági rések bármelyikének kihasználásához a támadónak hitelesített felhasználónak kell lenni.

Ez a biztonsági frissítés fontosnak minősül a Windows Server 2003, a Windows Server 2003 x64 Edition, a Windows Server 2008 és Windows Server 2008 x64 Edition érintett kiadásai vonatkozásában. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági frissítés a biztonsági réseket úgy küszöböli ki, hogy javítja az ADFS-képes webkiszolgálókon a webes ügyfél által küldött kérelmek fejrészének az ellenőrzését. A biztonsági résekkel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.**  A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft a minél előbbi telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** A[Microsoft Tudásbázis 971726. számú cikkében](http://support.microsoft.com/kb/971726) olvasható, hogy a felhasználók a biztonsági frissítés telepítésekor milyen ismert problémákkal találkozhatnak. A cikkben a problémákra javasolt megoldások is szerepelnek.

**Miért tartalmazza több bejelentett biztonsági rés javítását is ez a frissítés?**  
A frissítés több biztonsági rés javítását is tartalmazza, mivel a problémák megoldásához szükséges módosítások egymáshoz kapcsolódó fájlokban találhatók. A különálló, de egymással majdnem pontosan megegyező frissítések telepítése helyett így a vásárlóknak elegendő ezt az egy frissítést telepíteni.

**Mik az Active Directory összevonási szolgáltatásai?**  
Az Active Directory összevonási szolgáltatásai (ADFS) a WS-\* specifikációkban definiált, iparágilag támogatott, egyre szélesebb körben kibontakozó webszolgáltatási architektúrán alapulnak. Az ADFS segítségével az egyszeri bejelentkezési (SSO) mechanizmus révén hitelesíthetők a felhasználók több, egymással összefüggő webalkalmazás használatára az egyszeri bejelentkezési munkamenet teljes tartamára. Ehhez az ADFS lehetővé teszi a digitális identitások és jogcímek biztonsági eszközökön és vállalatokon átnyúló biztonságos megosztását.

**Hogyan ellenőrizhetem, hogy az ADFS is telepítve van-e a kiszolgálóra? **
Az ADFS telepítésének ellenőrzéséhez kattintson a **Start**, **Vezérlőpult**, majd a **Programok telepítése/törlése** elemre, duplán kattintson a **Windows-összetevők** pontra, és keresse a **Szövetségi szolgáltatások** elemet a telepített összetevők között.

**Mi a Windows Server 2003 R2?**  
A Windows Server 2003 R2 a Windows Server 2003 operációs rendszert egészíti ki úgy, hogy hatékonyabb módon kezeli és szabályozza a helyi és távoli erőforrások elérését, egyszerűen a létező Windows Server 2003 környezetbe integráltan. A Windows Server 2003 R2 bővíthető, fokozott biztonságú webes platformot és zavartalan együttműködést biztosít az UNIX alapú rendszerekkel, lehetővé tesz továbbá olyan új forgatókönyveket, mint az egyszerűsített csoportos kiszolgálókezelés, az identitások és hozzáférések javított kezelése, valamint a hatékonyabb tároláskezelés. A Windows Server 2003 R2 Datacenter Edition és a Windows Server 2003 R2 Enterprise Edition új, dinamikus licencelési módja lehetővé teszi, hogy a vevők még jobban kihasználják a kiszolgáló virtualizálásából keletkező értéket.

A Windows Server 2003 R2 vonatkozású további tájékoztatás a [Windows Server 2003 R2](http://www.microsoft.com/windowsserver2003/default.mspx) webhelyén található.

**Mik a WS-\* specifikációk?**

Ezek a webes szolgáltatások architektúrájára vonatkozó olyan specifikációk, amelyek alapja olyan ipari szabványok, mint a Simple Object Access Protocol (SOAP); az XML; a Web Service Description Language (WSDL) és az Universal Description, Discovery, and Integration (UDDI). A WS-\* biztosítja az alapot a földrajzilag is kiterjedt vállalatok részére a teljes, összehangolható üzleti megoldásokat, beleértve az identitások és a biztonság összevont kezelését is.

A webes szolgáltatási modell azon az elképzelésen alapul, hogy a vállalat szoftverrendszerei különböző nyelven íródtak, különböző programozási modell szerint, és sok különböző eszközről használják, illetve különböző típusú eszközökön futtatják. A webes szolgáltatások jelentik azt az eszközt, amelyik az elosztottan működő rendszerek összekapcsolását és együttműködését az internet igénybe vételével egyszerűen és hatékonyan biztosítja, függetlenül attól, hogy azok milyen nyelvre és platformra készültek.

Az egyszeri bejelentkezés meghamisítására lehetőséget adó biztonsági rés az ADFS biztonsági résén belül - CVE-2009-2508:  
------------------------------------------------------------------------------------------------------------------------
  
<span></span>  
A hamisítást lehetővé tévő biztonsági rés az Active Directory összevonási szolgáltatásaiban lehetővé teheti, hogy a támadó egy jogosult felhasználót személyesítsen meg, ha a támadónak hozzáférése van ahhoz a munkaállomáshoz és webböngészőhöz, amelyiket a kiszemelt felhasználó az egyszeri bejelentkezést kínáló webhely elérésére nemrég használt.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### A hibát enyhítő tényezők az ADFS biztonsági résén belül az egyszeri bejelentkezés meghamisítására lehetőséget adó biztonsági réshez - CVE-2009-2508
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   A támadó nem tudhatja kit kell megszemélyesíteni. A támadó csak azt a valakit tudja megszemélyesíteni, akinek a hitelesítési tokenjét meg tudta szerezni, például egy internetes fülkében.  
-   A támadás csak a munkamenetnek a kiszolgálón történő időtúllépése előtt lehetséges. Alapértelmezetten a munkamenetek a kiszolgálón 600 percig maradnak nyitva. Ha a támadó ez idő lejárta után próbálja meg az áldozat hitelesítési kódjának a használatát, a támadás sikertelen lesz.  
-   A támadónak hozzáféréssel kell rendelkezni az adott terminált előzőleg használó felhasználó hitelesítési tokenjéhez. A HTTPS protokollt használó kiszolgáló esetén ez a hitelesítési token az átvitel során védett, és a felhasználónak hozzáféréssel kell rendelkezni az áldozat számítógépéhez (például az internetes fülkében), hogy kihasználhassa a biztonsági rést. Az internetes fülkében a felhasználó bejelentkezhetett az ADFS egyszeri bejelentkezési szolgáltatását (SSO) használó webalkalmazásba, majd kijelentkezett az alkalmazásból. A támadó ekkor használhatja a számítógépet a webalkalmazás elérésére az előző felhasználó nevében, még akkor is, ha az a felhasználó kijelentkezett.
  
#### Lehetséges megoldások az ADFS biztonsági résén belül az egyszeri bejelentkezés meghamisítására lehetőséget adó biztonsági rés elkerüléséhez - CVE-2009-2508
  
A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### Gyakori kérdések az ADFS biztonsági résén belül az egyszeri bejelentkezés meghamisítására lehetőséget adó biztonsági résről - CVE-2009-2508
  
**Mire terjed ki a biztonsági rés hatása?**    
Ez a biztonsági rés tartalomhamisításra ad lehetőséget. A biztonsági rést sikeresen kihasználó támadó megszemélyesíthet más felhasználót, ennek eredményeként lehetőséget kap a felhasználó nevében az olyan webhely elérésére, amelyik az Active Directory összevonási szolgáltatásait (ADFS) használja az egyszeri bejelentkezéshez.
  
**Mi okozza a biztonsági rést?**    
Ez a biztonsági rés azért létezik, mert az ADFS nem eléggé biztosítja a munkamenet-kezelés ellenőrzését.
  
**Mire használhatja a biztonsági rést a támadó?**    
A közös vagy nyilvános használatú számítógépek esetén, mint az internetes fülkék, a biztonsági rést sikeresen kihasználó támadó elérheti más, előzőleg hitelesített felhasználó erőforrásait. Ez a támadás csak az ADFS rendszergazdája által konfigurált, alapértelmezetten 600 perces munkameneti időtúllépésen belül hajtható végre.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A támadó ezt a biztonsági rést a böngésző gyorsítótárában lévő specifikus adatok elérésével, majd azoknak az ADFS egyszeri bejelentkezési (SSO) szolgáltatását használó webes alkalmazásoknál a hitelesítésre történő felhasználásával tudja kihasználni. Miután a kiszolgálón a munkamenet időtúllépésbe került, az ADFS letiltja a kiszolgálói oldalon a hitelesített munkamenetet, és nem engedi, hogy a támadó újra használja ezeket a hitelesítési adatokat.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
Ez a biztonsági rés az ADFS egyszeri bejelentkezési (SSO) szolgáltatását használó webes kiszolgálókat érinti. A biztonsági rés nem érinti közvetlenül az ügyfélrendszereket.
  
**Mi a frissítés feladata?**    
A frissítés a biztonsági rést úgy szünteti meg, hogy a hitelesítőadatokat megfelelően megsemmisíti, amikor a felhasználó kijelentkezik a hálózatból.

Távoli kódfuttatás az ADFS biztonsági rése miatt - CVE-2009-2509  
----------------------------------------------------------------
  
<span></span>  
A távoli kódfuttatást lehetővé tévő biztonsági rés az Active Directory összevonási szolgáltatásának megvalósításában található. A biztonsági rés oka a kérelmek fejrészének helytelen visszaigazolása, amikor egy hitelesített felhasználó az ADFS szolgáltatást használó webes kiszolgálóhoz csatlakozik. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).
  
#### Az ADFS biztonsági rése miatt távoli kódfuttatást okozó biztonsági rés (CVE-2009-2509) hibáit enyhítő tényezők
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   A támadó csak érvényes bejelentkezési adatok birtokában használhatja ki a biztonsági réssel rendelkező kiszolgálót. A támadás csak azután hajtható végre, hogy az ADFS szolgáltatást támogató kiszolgálón sikeres volt a hitelesítés.
  
#### Az ADFS biztonsági rése miatt távoli kódfuttatást okozó biztonsági rés (CVE-2009-2509) hibáit elkerülő megoldások
  
A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### Az ADFS biztonsági rése miatt távoli kódfuttatást okozó biztonsági résre (CVE-2009-2509) vonatkozó gyakori kérdések
  
**Mire terjed ki a biztonsági rés hatása?**    
A biztonsági rés távolról történő kódfuttatást tehet lehetővé. A biztonsági rést sikeresen kihasználó támadó ezután WPI-jogosultsággal végezhet műveleteket az IIS-kiszolgálón. Ez alapértelmezésben azonos a Hálózati szolgáltatás fiók jogosultságaival.
  
**Mi okozza a biztonsági rést?**    
Ez a biztonsági rés azért létezik, mert az ADFS nem megfelelően érvényesíti a hitelesített ügyfelektől kapott kérelmek fejrészét.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó ezután WPI-jogosultsággal végezhet műveleteket az IIS-kiszolgálón. A WPI alapbeállítása Hálózati szolgáltatás fiók jogosultságoknak megfelelő. A rendszergazdai jogosultságokkal működő fiókot használó WPI-vel konfigurált alkalmazáskészlettel és ASP-oldalakkal rendelkező IIS-kiszolgálókat komolyabban érintheti a biztonsági rés okozta probléma, mint azokat az IIS-kiszolgálókat, amelyek alapértelmezett WPI-beállításokkal konfigurált alkalmazáskészlettel rendelkeznek.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A támadó megpróbálhatja kihasználni a biztonsági rést úgy, hogy speciálisan erre a célra kialakított HTTP-kérést küld az ADFS szolgáltatást használó webkiszolgálónak. A biztonsági rés kihasználásához a támadónak hitelesített felhasználónak kell lenni.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
Az ADFS szolgáltatást használó webkiszolgálók veszélyeztetettek. Az ügyfélrendszereket nem érinti ez a biztonsági rés.
  
**Mi a frissítés feladata?**    
Ez a frissítés kijavítja az ADFS szolgáltatást használó webkiszolgálókon a webes ügyfél által küldött kérelmek fejrészének az érvényesítését.

Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről.
