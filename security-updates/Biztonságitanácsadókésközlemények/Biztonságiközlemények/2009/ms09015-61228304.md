Security Bulletin

A Microsoft számú biztonsági közleménye MS09-015 - Közepes
==========================================================

A SearchPath kevert fenyegetéses biztonsági rése jogok kiterjesztését teszi lehetővé (959426)
---------------------------------------------------------------------------------------------

Közzétéve: 2009. április 14. | Frissítve: 2009. április 15.

**Verzió:** 1.1

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Windows SearchPath funkció nyilvánosan jelentett biztonsági résére kínál megoldást, amely jogok kiterjesztését tette lehetővé, amennyiben a felhasználó speciálisan kialakított fájlt töltött le egy adott helyről, majd megnyitott egy olyan alkalmazást, amely bizonyos körülmények között letölthette ezt a fájlt.

A biztonsági frissítés mérsékelt besorolást kapot az összes támogatott Windows XP, Windows Server 2003, Windows Vista és Windows Server 2008 rendszer vonatkozásában. A Microsoft Windows 2000 rendszer esetében a frissítés a biztonsági rendszer lényeges módosításának minősül. Részleteket a biztonsági közlemény **gyakran ismételt kérdésekkel kapcsolatos** részében olvashat. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági frissítés azáltal oldja meg a biztonsági rés okozta problémát, hogy módosítja a fájlok Asztalról történő betöltési módját a Windows rendszerben. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

A biztonsági frissítés az először a [953818. számú Microsoft biztonsági tanácsadóban](http://technet.microsoft.com/security/advisory/953818) ismertetett biztonsági rést is megszünteti.

**Javaslat.** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft az azonnali telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban
----------------------------------------------------------------

<span></span>
**Hol találhatók a fájlinformációs adatok?**  
A fájlinformációs adatok a [Microsoft Tudásbázis 959426. számú cikkében](http://support.microsoft.com/kb/959426) találhatók.

**Hogyan kapcsolódik a biztonsági frissítés az MS09-014 közleményben ismertetetthez?**  
A közlemény tárgyát képező, kevert fenyegetés távoli kódfuttatást okozó biztonsági rése ([CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540)) az Internet Explorer alkalmazást is érinti. A Microsoft azt javasolja, hogy telepítsék az [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659) biztonsági közleményben található biztonsági frissítést: Összesítő biztonsági frissítés az Internet Explorer programhoz (963027).

**Tartalmaz a frissítés funkcionalitásbeli módosítást is?**  
Igen. A frissítés **A biztonsági rés részletes ismertetése** című szakaszban ismertetett módosítások mellett olyan hatékony védelmi módosítást tartalmaz, amely új API-elemet vezet be (SetSearchPathMode):

**A SetSearchPathMode hatékony védelmi módosítás mire ad lehetőséget?**  
Az új API feladatonkénti üzemmód használatát teszi lehetővé a SearchPath funkcióval történő fájlkereséshez. Ezáltal az alkalmazások az aktuális könyvtárban való keresésre kényszeríthetik a rendszert az alkalmazás- és rendszerhelyeket követően.

**Ha csak alkalmazom a biztonsági frissítést, rendszerem teljesen védett lesz a CVE-2008-2540 azonosítóval ellátott kevert fenyegetési probléma ellen?**  
Nem, külön engedélyezni kell az Internet Explorer összesítő biztonsági frissítésében foglalt hatékony védelmet azon felül, hogy alkalmazza az [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659) közleményben foglaltakat. Ellenőrizni kell azonban, hogy a kevert fenyegetés elleni hatékony védelem engedélyezése megfelelő-e az aktuális rendszerkörnyezetben.

Az Internet Explorer hatékony védelmi funkciójának engedélyezéséhez aktiválja a SearchPath szolgáltatásvezérlő kulcsot az [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659) közlemény Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban részében leírt módon. A kevert fenyegetési probléma részleteit lásd az [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) normál biztonsági rés besorolás azonosítónál.

**A Microsoft Windows 2000 miért a biztonsági hatás és a súlyosság fokának megjelölése nélkül szerepel az érintett szoftverek táblázatában?**
A Microsoft Windows 2000 azért szerepel a biztonsági hatás és a súlyosság fokának megjelölése nélkül, mert a frissítés nem tartalmaz ugyanolyan javítást a Microsoft Windows 2000 rendszerre nézve, mint a többi támogatott Windows operációs rendszerhez. A jelen frissítés csak egy hatékony védelmi módosítást kínál a Microsoft Windows 2000 rendszerhez.

**A Microsoft Windows 2000 rendszerben lévő kevert fenyegetési problémára miért más javítás vonatkozik, mint a többi platformra? **
A Microsoft Windows 2000 rendszerben nincs meg a SetSearchPathMode funkció alkalmazásához szükséges architektúra, ezért a biztonsági rés kiszűrése nem oldható meg. Ehhez a Microsoft Windows 2000 jelentős részét át kellene alakítani – nem csak az érintett összetevő architektúráját. Ennek eredményeképpen a Microsoft Windows 2000 kompatibilitása nem lesz megfelelő: nincs rá garancia, hogy a Microsoft Windows 2000 rendszerben történő futtatásra tervezett alkalmazások továbbra is működnek rajta. A Microsoft azonban nem ismer kifejezetten a Microsoft Windows 2000 rendszer biztonsági rését kihasználó konkrét támadási felületet.

Az architektúra módosításai miatt a Microsoft Windows 2000 rendszerben engedélyezni kell az erre a rendszerre vonatkozó hatékony védelmi módosítást a HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode beállításkulcs használatával. Tudnivalókkal kapcsolatban tekintse meg a [Microsoft Tudásbázis 959426 számú cikkét](http://support.microsoft.com/kb/959426). Tudnivalókkal kapcsolatban tekintse meg az MSDN cikkét: [Dinamikus csatolású függvénytár keresési sorrendje](http://msdn.microsoft.com/en-us/library/ms682586.aspx).

**Mi a SearchPath funkció?**  
A SearchPath funkció adott fájl megkeresésére szolgál egy ismert elérési útvonalon. További információért olvassa el a [SearchPath funkció](http://msdn.microsoft.com/en-us/library/aa365527.aspx) című MSDN-cikket.

**Mi a LoadLibrary funkció?**  
A LoadLibrary funkció betölti a megadott modult a hívó folyamatban megadott címtérbe. Ez a modul más modulok betöltését is okozhatja. További információért olvassa el a [SearchPath funkció](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx) című MSDN-cikket.

**A biztonsági közleményben tárgyalt szoftvernél korábbi kiadással rendelkezem. Mi a teendő?**  
Teszteléssel állapították meg a közleményben tárgyalt szoftverkiadások érintettségét. A többi kiadás támogatási életciklusa végére ért. Az egyes szoftverkiadások támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.

A szoftver korábbi kiadásai felhasználóinak számára azt javasoljuk, hogy a potenciális biztonsági veszélyek elkerülése érdekében sürgősen térjenek át valamelyik támogatott kiadású szoftverre. Ha többet szeretne tudni a Windows termékek életciklusáról, keresse fel a [Microsoft támogatási életciklusokkal foglalkozó webhelyét](http://go.microsoft.com/fwlink/?linkid=21742). Ha többet szeretne tudni az ilyen szoftververziók illetve kiadások biztonsági frissítéseire vonatkozó meghosszabbított terméktámogatási időszakokról, keresse fel a [Microsoft terméktámogatási szolgálatát](http://go.microsoft.com/fwlink/?linkid=33328).

Kevert fenyegetés jogok kiterjesztését okozó biztonsági rése - CVE-2008-2540  
----------------------------------------------------------------------------
  
<span></span>
A kevert fenyegetés jogok kiterjesztését okozó biztonsági rése abban áll, ahogyan a Windows SearchPath funkciója azonosítja és megnyitja a fájlokat a rendszerben. A támadónak a biztonsági rés kihasználásához meg kell győznie a felhasználót, hogy letöltsön egy speciálisan kialakított fájlt egy adott helyre, majd megnyisson egy alkalmazást, amely bizonyos körülmények között betöltheti a fájlt.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).

#### Gyakran feltett kérdések a kevert fenyegetés jogok kiterjesztését okozó biztonsági rése (CVE-2008-2540) kapcsán
  
**Mire terjed ki a biztonsági rés hatása?**    
A kevert fenyegetést tartalmazó biztonsági rés jogok kiterjesztését okozhatja. Ha egy támadó sikeresen kihasználja a biztonsági rést, teljes mértékben átveheti az érintett rendszer távoli irányítását. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.
  
**Mi okozza a biztonsági rést?**    
Az Internet Explorer a Windows alkalmazáskönyvtárainak elhelyezkedése miatt az Asztalról képes megnyitni egy különlegesen kialakított fájlt. Az összesített (kevert) fenyegetés során a fájlokat a rendszer kérdés nélkül tölti le a rendszerre, így az Internet Explorer a szándék ellenében a fájlt az Asztalról, és nem pedig a Windows rendszerből tölti be.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó mesterséges kódot futtathat az érintett rendszeren. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A támadó speciálisan kialakított fájlt készt, majd ráveheti a felhasználót, hogy letöltse azt az Asztalra. Ezután el kell indítani egy alkalmazást a speciálisan kialakított fájl megnyitásához. A támadó azonban nem tudja rákényszeríteni a felhasználókat, hogy letöltsék az ilyen fájlokat és az Asztalra helyezzék azokat.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
Minden olyan rendszert érint a biztonsági rés (például munkaállomások és terminálkiszolgálók), amelyben a felhasználók fájlokat töltenek le és alkalmazásokat indítanak.
  
**Mi a frissítés feladata?**    
A frissítés módosítja a keresési sorrendet az alkalmazáskönyvtárakban (például a rendszerkönyvtárban keres először, és csak azután az aktuális munkakönyvtárakban). A frissítés hatékony védelmi módosítás gyanánt új, SetSearchPathMode elnevezésű API-elemet kínál fel a haladó felhasználóknak, akik tetszés szerinti keresési sorrendet állíthatnak be a SearchPath funkcióban. A hatékony védelmi megoldás részleteit lásd a **Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban** c. részben.
  
**Miért kapta ez a biztonsági rés és a kevert fenyegetés távoli kódfuttatást okozó biztonsági rése ugyanazt a CVE-azonosítót (CVE-2008-2540) az MS09-014 jelzésű, az Internet Explorer összesítő frissítésében (963027)?**  
[A CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) jelzésű probléma a Windows XP Service Pack 2, Windows XP Service Pack 3 rendszerek Internet Explorer 7 alkalmazásait, valamint a Windows XP Professional x64 Edition és Windows XP Professional x64 Edition Service Pack 2 rendszereket, továbbá a Windows SearchPath funkcióját is érinti. A Microsoft azt javasolja, hogy telepítsék az [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659) biztonsági közleményben található biztonsági frissítést: Összesítő biztonsági frissítés az Internet Explorer programhoz (963027). A frissítés hatékony védelmi módosítása, a SetSearchPathMode alkalmazásprogramozási felület (API), megfelel az [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659) közlemény javításának. Lásd még a Tartalmaz a frissítés funkcionalitásbeli módosítást is? című szakaszt a **Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban** című részben.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**    
Igen. A biztonsági rés nyilvánosságra került, A normál biztonsági rés besorolást és a [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) azonosítót kapta. A biztonsági frissítés egy, elsőként a [Microsoft 953818. számú biztonsági tanácsadója](http://technet.microsoft.com/security/advisory/953818) ismertetett biztonsági résre is kitér.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**    
Nem. E közlemény eredeti verziójának kiadásáig a Microsoft nem kapott olyan információt, amely arra utalt volna, hogy ezt a biztonsági rést nyilvános támadásra használták, és nem találkozott olyan nyilvánosságra hozott leírásokkal, amelyek tartalmazták volna egy sikeres támadás forgatókönyvét.

**A frissítés telepítésének ellenőrzése**

-   **Microsoft Baseline Security Analyzer**

    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        **Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        **Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        **Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        **Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.
