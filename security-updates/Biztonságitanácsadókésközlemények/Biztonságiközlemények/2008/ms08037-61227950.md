Security Bulletin

A Microsoft számú biztonsági közleménye MS08-037 - Fontos
=========================================================

A DNS szolgáltatás tartalomhamisítást lehetővé tévő biztonsági rései (953230)
-----------------------------------------------------------------------------

Közzétéve: 2008. július 8. | Frissítve: 2009. december 9.

**Verzió:** 3.1

### Általános tudnivalók

#### Összefoglalás

Ez a biztonsági frissítés a Windows Domain Name System (DNS) rendszerben tartalomhamisítást lehetővé tévő két közvetlenül jelzett biztonsági rést szüntet meg. Ezek a biztonsági rések megtalálhatók a DNS ügyfél és kiszolgálói változatában is, és lehetővé teszik, hogy az interneten lévő rendszereknek szánt hálózati forgalmat a támadó saját rendszerére irányítsák át.

A Windows 2000, a Windows XP, a Windows Server 2003 és a Windows Server 2008 rendszerek összes támogatott kiadására nézve ez a biztonsági frissítés fontos besorolásúnak minősül. További tudnivalókért tekintse meg az **Érintett és nem érintett szoftverek** c. részt.

A biztonsági frissítés a rések kezeléséhez erősen véletlenszerű DNS tranzakcióazonosítókat, az UDP lekérdezésekhez véletlenszerű szoftvercsatornákat használ, és frissíti a DNS gyorsítótár kezeléséhez használt algoritmust. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.** A Microsoft a biztonsági frissítés lehető leghamarabb történő telepítését javasolja ügyfeleinek.

**Ismert problémák** A [Microsoft Tudásbázis 953230. cikkében](http://support.microsoft.com/kb/953230/hu) olvasható, hogy a felhasználók a biztonsági frissítés telepítésekor milyen, jelenleg már ismert problémákkal találkozhatnak.

#### Érintett és nem érintett szoftverek

Az itt felsorolt szoftvereket a verziók és változatok érintettségének felderítése érdekében teszteltük. A többi verzió, illetve kiadás támogatási életciklusa végére ért vagy nem érintett. Az egyes szoftverek és azok verzióinak támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.

Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban  
----------------------------------------------------------------
  
<span></span>
**Miért adták ki újra a közleményt 2009. december 8-án?**    
A Microsoft az újra kiadott közleményben felajánlja a Microsoft Windows 2000 Service Pack 4 DNS-ügyfelének frissítését (KB951748), hogy erősen véletlenszerű DNS tranzakció-azonosítókat biztosítson a további parancselérési utakhoz. A többi Windows platformtól eltérően a Microsoft Windows 2000 két parancselérési utat használ a DNS tranzakciókhoz. A korábbi frissítés csak egy parancselérési úthoz biztosította a tranzakció-azonosítók véletlen generálását. A frissítés új kiadása a Microsoft Windows 2000 rendszerben a másik parancselérési úthoz is biztosítja a tranzakció-azonosítók véletlen generálását. A Microsoft Windows 2000 Service Pack 4 DNS-ügyfél frissítését (KB951748) előzőleg már telepítő felhasználóknak ismét telepíteni kell az újra automatikusan felajánlott frissítést. Az újrakiadás nem érint más frissítést.
  
**Miért módosították a biztonsági közleményt 2009. január 13-án?**  
A Microsoft a módosított közleményben ismerteti, hogy a közlemény által felajánlott frissítést nem minden Windows XP Service Pack 3 rendszerhez ajánlották fel helyesen. Az észlelési és telepítési problémát javították, és a Windows XP Service Pack 3 rendszerek azon felhasználóinak, akik még nem alkalmazták a közleményben lévő frissítést, mostantól helyesen ajánlják fel a frissítést. A frissítéshez kapcsolódó bináris fájlok nem változtak. A rendszerüket sikeresen frissített felhasználóknak nem kell újra telepíteniük a frissítést.
  
**Miért módosították a biztonsági közleményt 2008. július 25-én?**  
A Microsoft módosította a biztonsági közleményt, amelyben most már a **biztonsági frissítéssel foglalkozó gyakran feltett kérdéseket tartalmazó rész három új problémával** bővült. Ezek a gyakran feltett kérdések a következők:
  
-   **A frissítés telepítését követően milyen portütközési problémák tapasztalhatóak?**  
    Azok a DNS-kiszolgálók, amelyeknek a DNS-kiszolgáló által a csatornatartományhoz kiosztott porttartományon belül van szükségük statikus UDP-portokra, elképzelhető, hogy szolgáltatási problémákhoz vezető portütközéseket tapasztalnak majd. A biztonsági frissítés telepítésével kapcsolatos jelenleg ismert problémákról a [Microsoft Tudásbázis 953230 számú](http://support.microsoft.com/kb/953230/hu) cikkében olvashatnak bővebben.  
-   **A frissítés telepítését követően az SBS-felhasználóknál milyen problémák tapasztalhatóak?**  
    A Windows Small Business Server (SBS) 2003 támogatott verziói ugyanazt az érintett kódot tartalmazzák, mint a Windows Server 2003 Service Pack 1 és a Windows Server 2003 Service Pack 2. Az SBS-rendszereknél azonban, a közlemény által terjesztett frissítés telepítése után szolgáltatási problémák léphetnek fel. A biztonsági frissítés telepítésével kapcsolatos jelenleg ismert problémákról a [Microsoft Tudásbázis 953230 számú](http://support.microsoft.com/kb/953230/hu) cikkében olvashatnak bővebben.  
-   **A telepítést követően a külső tűzfalaknál és külső NAT-eszközökön milyen problémák léphetnek fel?**  
    A közlemény által ajánlott frissítés Windows rendszerrel működő számítógépre való telepítését követően, lehetséges, hogy az arról a gazdagépről származó DNS-lekérdezések a tűzfalon keresztül nem fognak véletlenszerűen forrásportokat használni. A biztonsági frissítés telepítésével kapcsolatos jelenleg ismert problémákról a [Microsoft Tudásbázis 953230 számú](http://support.microsoft.com/kb/953230/hu) cikkében olvashatnak bővebben.
  
**Miért módosították a biztonsági közleményt 2008. július 23-án?**  
A Microsoft módosította a biztonsági közleményt, így most már a frissítés által felváltottként tartalmazza az MS06-064, MS07-062 és a MS08-001 közleményeket.
  
**Miért módosították a biztonsági közleményt 2008. július 10-én?**  
A Microsoft módosította a biztonsági közleményt, annak érdekében, hogy a ZoneAlarm és a Check Point Endpoint Security (korábban Check Point Integrity néven ismert) [(Check Point Software Technologies Ltd.)](http://www.checkpoint.com/) felhasználói tájékoztatást nyerhessenek egy internetkapcsolati problémára vonatkozóan. A probléma ismertetése a gyakran feltett kérdések között olvasható. A módosítás nem változtatta meg a közleményhez tartozó fájlokat. A frissítést már telepített felhasználóknak nem kell még egyszer telepíteniük azt. A Check Point itt feltüntetett szoftverének felhasználói számára javasolt, hogy ismerkedjenek meg a gyakran feltett kérdések között írottakkal.
  
**Milyen tüneteket tapasztalhatnak ZoneAlarm és Check Point Endpoint Security felhasználói a frissítés telepítését követően?**  
A Microsoft tudomására jutott, hogy a ZoneAlarm és a Check Point Software Technologies Ltd. által gyártott Check Point Endpoint Security (korábban Check Point Integrity néven ismert) szoftver felhasználói az MS08-037 biztonsági közlemény által felajánlott frissítések telepítése után internetkapcsolati problémákkal szembesülnek. A Microsoft Check Point céggel együtt továbbra is folytatja a probléma feltárását. A Microsoft javasolja a [ZoneAlar](http://download.zonealarm.com/bin/free/pressreleases/2008/lossofinternetaccessissue.html)m és a Check Point Endpoint Security felhasználóinak, hogy látogassanak el a megfelelő [Check Point-weboldalra](https://supportcenter.checkpoint.com/supportcenter/index.jsp), valamint ismerkedjenek meg a biztonsági közlemény útmutatásaival, és az esetleges új szoftverekkel is.
  
A Microsoft a külső felek elérhetőségi adatait a technikai támogatás eléréséhez szánt segítségnyújtásként teszi közzé. A kapcsolattartási információ értesítés nélkül változhat. A Microsoft nem garantálja ezen felek elérhetőségi adatainak pontosságát. A cikkben említett, harmadik féltől származó termékek a Microsofttól független gyártók termékei. A Microsoft sem közvetve, sem más módon nem vállal garanciát a termékek teljesítményére vagy megbízhatóságára vonatkozóan.
  
**Milyen telepítéseltávolítási problémákkal találkozhatnak a felhasználók a biztonsági frissítések telepítésekor?**  
A Microsoft Windows 2000 és Windows 2003 támogatott kiadásaihoz DNS-kiszolgáló- és DNS-ügyfélfrissítéseket egyaránt felajánl a rendszer. A DNS-kiszolgáló- és az -ügyfélfrissítések bináris értékei közösek. Eltávolításuk a telepítéssel ellentétes sorrendben történjen, nehogy a megosztott bináris értékek visszaálljanak egy korábbi verzióra. A [Microsoft Tudásbázis 823836. cikkében](http://support.microsoft.com/kb/823836/hu) olvasható, hogy a felhasználók a biztonsági frissítések eltávolításakor milyen, jelenleg már ismert problémákkal találkozhatnak.
  
**A véletlenszerű szoftvercsatorna-kapcsolathoz kötődő milyen problémákkal találkozhatnak a felhasználók a biztonsági frissítés telepítésekor?**  
Alapértelmezés szerint a jelen biztonsági közleménnyel ajánlott DNS-frissítések maximálisan kihasználják az igénybe vehető szoftvercsatornákat, így nagyobb entrópiát biztosítanak. Ha azonban a felhasználó porttartományt határozott meg a beállításjegyzékben, akkor a frissítések figyelembe veszik a felhasználó beállítását, és csak az ott megadott csatornákat fogják igénybe venni.
  
A tartományok a beállításjegyzék következő helyén állíthatók be:  
**HKLM\\System\\CurrentControlSet\\Services\\DNS\\Parameters Reg key Name: SocketPoolSize**  
**Megjegyzés** A változások életbeléptetéséhez újra kell indítani a DNS szolgáltatást.
  
**Mit jelent csatornatartományt megadása?**  
Az azonos csatornatartományt használó más alkalmazásokkal vagy szolgáltatásokkal való ütközés elkerülésére esetleg be kell állítani a DNS számára választható csatornatartományt. A MaxUserPort beállításkulcs jelentése más a Windows Vista és Windows Server 2008 rendszerben, és más a Microsoft Windows Server 2000 és Windows Server 2003 rendszerben. A [Microsoft Tudásbázis 929851. sz. cikke](http://support.microsoft.com/kb/929851/hu) részletezi a Windows Vista és a Windows Server 2008 rendszer működése közti különbségeket ebben a tekintetben.

A porttartományok lefoglalásának részleteit a Microsoft Windows 2000 Server és Windows Server 2003 rendszerben lásd a [Microsoft Tudásbázis 812873. sz. cikkét](http://support.microsoft.com/kb/812873/hu).

A DNS elégtelen szoftvercsatorna-entrópiájának biztonsági rése - CVE-2008-1447  
------------------------------------------------------------------------------
  
<span></span>
A tartalomhamisítást lehetővé tevő biztonsági rés létezik a Windows DNS-ügyfélben és a Windows DNS-kiszolgálóban is. Ez a biztonsági rés lehetővé tette, hogy távoli jogosulatlan támadó gyorsan és meggyőzően hamisítsa a válaszokat, és a DNS kiszolgálói vagy ügyféli gyorsítótárába olyan tartalmat szúrjon be, amelyik átirányítja az internetes forgalmat.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2008-1447](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1447).
  
#### A DNS elégtelen szoftvercsatorna-entrópiájának biztonsági résének (CVE-2008-1447) súlyosságát enyhítő tényezők
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét.
  
-   A TCP és az IP réteg felett működő titkosító protokollok, például az IPsec és az SSL/TLS, megakadályozhatják, hogy a támadó figyelhesse vagy átirányíthassa a forgalmat.
  
#### A DNS elégtelen szoftvercsatorna-entrópiájának biztonsági rését (CVE-2008-1447) kiküszöbölő lehetőségek
  
A lehetséges megoldás olyan beállítást vagy konfigurációs módosítást takar, amely magát a biztonsági rést nem szünteti meg, azonban a frissítés telepítése előtt segít az ismert támadási lehetőségek kivédésében.
  
-   A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### Gyakori kérdések a DNS elégtelen szoftvercsatorna-entrópiájának biztonsági réséről - CVE-2008-1447
  
**Mire terjed ki a biztonsági rés hatása?**    
A tartalomhamisítást lehetővé tevő biztonsági rés létezik a Windows DNS-ügyfélben és a DNS-kiszolgálóban is. A biztonsági rést sikeresen kihasználó támadó tetszőleges címeket szúrhat be a DNS gyorsítótárába.
  
**Mi okozza a biztonsági rést?**    
A Windows DNS ügyfél, illetve kiszolgálói rendszerében lévő Windows DNS szolgáltatás nem biztosít elegendő entrópiát a DNS lekérdezések végrehajtásakor.
  
**Mi a Tartománynévrendszer (DNS)?**  
A Tartománynévrendszer (DNS) a TCP/IP-rendszert alkotó, iparági szabványoknak megfelelő protokollcsomagok egyike. A DNS két szoftverösszetevő segítségével működik: ez a DNS-kiszolgáló és a DNS-ügyfél (azaz a feloldó). Mindkét összetevő a háttérben futó szervizalkalmazás. A hálózati erőforrásokat számokból álló IP-címek azonosítják. A felhasználók azonban nehezen jegyzik meg ezeket. A DNS adatbázisa olyan rekordokat tartalmaz, amelyek a hálózati erőforrások olyan felhasználóbarát alfanumerikus nevét, mint például a www.microsoft.com, a kommunikációs célokra alkalmas IP-címmé képezi le. A DNS ilyen értelemben tehát emlékeztető eszköz, amely megkönnyíti a hálózati erőforrások megjegyzését a felhasználók számára. További információkért, valamint a DNS és a többi Windows-technológia együttműködését bemutató logikai ábrákat lásd a [Mi a DNS?](http://technet2.microsoft.com/windowsserver/en/library/ff937311-03ce-4d04-b72c-b39c4d51cb361033.mspx) c. cikkben.
  
**Mi a DNS gyorsítótár?**  
A tartománynév-kezelő rendszer (Domain Name System - DNS) gyorsítótára olyan szolgáltatás, amelyik tárolja a DNS lekérdezésre adott válaszokat, hogy ugyanazokat az adatokat ne kelljen ismételten lekérdezni a DNS-kiszolgálótól. További tájékoztatást a Technet [DNSCache](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/30643.mspx?mfr=true) és a [DNS Resolver Cache Service](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/cnet/cnbc_imp_qxht.mspx?mfr=true) című cikke tartalmaz. A DNS "mérgezéséről" a további tájékoztatást a Technet [Attack detection](http://www.microsoft.com/technet/isa/2004/help/fw_alertattack.mspx?mfr=true) (Támadás észlelése) című cikke tartalmaz.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó tetszőleges címeket szúrhat be a DNS gyorsítótárába, ezt szokták nevezni a DNS mérgezésének.
  
**Hogyan használhatja ki a támadó a biztonsági rést? **  
A támadó a veszélyeztetett DNS-kiszolgálónak vagy ügyfélnek küldhetett bizonyos lekéréseket, és ugyanakkor olyan módon válaszol, ami lehetővé teszi, hogy a támadó hamis vagy félrevezető DNS adatokat szúrjon be. A támadó ezt követően átirányíthatj az internernetes forgalmat a jogszerű helyekről a támadó által megválasztott címre.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
Minden, hálózathoz vagy internethez csatlakozó Windows-rendszer veszélyeztetett. A Windows Vista és a Windows Server 2008 rendszert nem érinti a biztonsági rés.
  
**Mi a frissítés feladata? **  
A frissítés ezt a biztonsági rést úgy szünteti meg, hogy a DNS távoli lekérdezéseihez erősen véletlenszerű tranzakcióazonosítókat és véletlenszerű UDP szoftvercsatornákat használ.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**    
Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**    
Nem. E közlemény eredeti verziójának kiadásáig a Microsoft nem kapott olyan információt, amely arra utalt volna, hogy ezt a biztonsági rést nyilvános támadásra használták, és nem találkozott olyan nyilvánosságra hozott leírásokkal, amelyek tartalmazták volna egy sikeres támadás forgatókönyvét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        **Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        **Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.

#### Tudnivalók a központi telepítésről
  
**A frissítés telepítése**
  
A biztonsági frissítés telepítésekor a telepítő ellenőrzi, hogy a számítógépen frissítendő fájlokat egy korábbi Windows-gyorsjavítás nem frissítette-e.
  
A közleményben alkalmazott szakkifejezésekkel (pl. *gyorsjavítás*) kapcsolatban tekintse meg a [Microsoft Tudásbázis 824684](http://support.microsoft.com/kb/824684/hu) számú cikkét.
  
A biztonsági frissítés telepítője az alábbi kapcsolók használatát támogatja:
  
| Kapcsoló          | Leírás                                                                                                                                                              |  
|-------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | A támogatott kapcsolóknál megjeleníti a súgót.                                                                                                                      |  
| **/quiet**        | Nem jeleníti meg az állapot- és hibaüzeneteket.                                                                                                                     |  
| **/norestart**    | A **/quiet** kapcsolóval együtt alkalmazva a rendszer telepítés után nem indul újra, még akkor sem, ha a telepítés befejezéséhez a rendszert újra kellene indítani. |
  
**Megjegyzés** A wusa.exe telepítőről további információt a [Microsoft tudásbázis 934307. számú cikkében](http://support.microsoft.com/kb/934307/hu) talál.
  
**A frissítés telepítésének ellenőrzése**
  
-   **Microsoft Baseline Security Analyzer**
  
    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.
  
-   **Fájlverziószám ellenőrzése**
  
    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.
  
    1.  Kattintson a **Start** elemre, majd a **Keresés indítása** mezőben adja meg a frissítőfájl nevét.  
    2.  Amikor a fájl megjelenik a **Programok** között, jobb egérgombbal kattintson a fájlnévre, majd a **Tulajdonságok** lehetőségre.  
    3.  Az **Általános** lapon hasonlítsa össze a fájlméretet a közlemény Tudásbázis cikkében ismertetett fájlinformációs táblázatokkal.  
    4.  Az adatok összevetéséhez (pl. fájlverzió, a módosítás dátuma) a **Részletek** lapra is kattinthat, a közlemény Tudásbázis cikkében ismertetett fájlinformációs táblázatok alapján.  
    5.  Kattinthat a **korábbi verziókat** felsoroló lapra is, majd összehasonlíthatja a korábbi fájlverzió adatait az aktuális, frissített fájléval.
