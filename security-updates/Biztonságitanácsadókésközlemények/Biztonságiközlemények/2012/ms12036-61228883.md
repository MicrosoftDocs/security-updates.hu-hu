Security Bulletin

A Microsoft számú biztonsági közleménye MS12-036 - Kritikus
===========================================================

A Távoli asztal biztonsági rése távoli kódfuttatást tehet lehetővé (2685939)
----------------------------------------------------------------------------

Közzétéve: 2012. június 12. | Frissítve: 2012. június 13.

**Verzió:** 1.1

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Remote Desktop Protocol közvetlenül jelzett biztonsági rését szünteti meg. A biztonsági rés távoli kódfuttatást tehet lehetővé, ha a támadó különlegesen kialakított RDP-csomagsorozatot küld az érintett rendszerre. Alapértelmezés szerint a Remote Desktop Protocol (RDP) protokoll nem engedélyezett egyik Windows operációs rendszeren sem. Azok a rendszerek, amelyeknél az RDP nem engedélyezett, nem veszélyeztettek.

A biztonsági frissítés a Windows Server 2003 és a Windows Server 2008 minden támogatott kiadására, a Windows 7 32 bites rendszerekhez Service Pack 1 és a Windows 7 x64-alapú rendszerekhez Service Pack 1 verziójára, valamint a Windows Server 2008 R2 minden támogatott kiadására nézve is Kritikus besorolású. A biztonsági frissítés emellett Mérsékelt besorolású a Windows XP és a Windows Vista minden támogatott kiadására, valamint a Windows 7 32 bites rendszerekhez és a Windows 7 x64-alapú rendszerekhez készült változatára nézve. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági frissítés a biztonsági rés kiküszöböléséhez módosítja azt az eljárást, amellyel a Remote Desktop Protocol feldolgozza a csomagokat a memóriában. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat:** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítéseket, a Microsoft az azonnali telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák.** Nincsenek

Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban
----------------------------------------------------------------

<span></span>
**Érintett operációs rendszert használok, de nem engedélyeztem a Remote Desktop Protocol (RDP) összetevőt. Így is felkínálják számomra a frissítést?**
Igen. Bár a biztonsági rések nem veszélyeztetik azokat a rendszereket, amelyeken nem engedélyezték az RDP protokollt, az ilyen rendszerek védelme érdekében az RDP protokollt nem engedélyező felhasználóknak is felajánljuk a frissítést.

**Hol találhatók a fájlinformációs adatok?**
A fájlinformációs adatokat lásd a **Biztonsági frissítés telepítése** rész hivatkozási táblázataiban.

**A biztonsági közleményben tárgyalt szoftvernél korábbi kiadással rendelkezem. Mi a teendő?**
Teszteléssel állapították meg a közleményben tárgyalt szoftverkiadások érintettségét. A többi kiadás támogatási életciklusa végére ért. Ha többet szeretne tudni a termékek életciklusáról, keresse fel a [Microsoft támogatási életciklusokkal](http://go.microsoft.com/fwlink/?linkid=21742)foglalkozó webhelyét.

A szoftver korábbi kiadásai felhasználóinak számára azt javasoljuk, hogy a potenciális biztonsági veszélyek elkerülése érdekében sürgősen térjenek át valamelyik támogatott kiadású szoftverre. A szoftverkiadás támogatási életciklusáról lásd a [Termék kiválasztása életciklusadatok megtekintéséhez](http://go.microsoft.com/fwlink/?linkid=169555) c. részt. A szoftverkiadásokhoz tartozó javítócsomagok részleteit lásd [A szervizcsomag támogató szolgáltatásának életciklusra vonatkozó új szabályzata](http://go.microsoft.com/fwlink/?linkid=89213) című részben.

Azok az ügyfelek, akik további támogatást igényelnek ezekhez a korábbi verziókhoz, a támogatási lehetőségekről érdeklődjenek a Microsoft ügyfélkapcsolati képviselőjénél, a műszaki kapcsolattartótól vagy a Microsoft megfelelő partnerképviseletétől. Alliance, Premier vagy Authorized szerződéssel rendelkező felhasználóink forduljanak a Microsoft helyi értékesítési irodájához. Az elérhetőségeket keresse a [Microsoft Worldwide Information webhelyen](http://go.microsoft.com/fwlink/?linkid=33329): válasszon egy országot a legördülő listából, majd az **Indítás** gombra kattintva jelenítse meg a telefonszámokat. Telefonáláskor keresse a helyi értékesítési menedzsert. További információ: [A Microsoft termékek terméktámogatási időszakával kapcsolatos, gyakran ismételt kérdések](http://go.microsoft.com/fwlink/?linkid=169557).

A Remote Desktop Protocol biztonsági rése – CVE-2012-0173
---------------------------------------------------------

<span></span>
A Remote Desktop Protocol összetevőnek a memóriában nem megfelelően inicializált vagy törölt objektumokhoz való hozzáférési módja távoli kódfuttatást okozhat. A biztonsági rést sikeresen kihasználó támadó tetszőleges kódot futtathat a célrendszeren. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.

A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2012-0173](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0173).

#### A Remote Desktop Protocol biztonsági résének (CVE-2012-0173) súlyosságát enyhítő tényezők

Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:

-   Alapértelmezés szerint a Remote Desktop Protocol nem engedélyezett egyik Windows operációs rendszeren sem. Azok a rendszerek, amelyeknél az RDP nem engedélyezett, nem veszélyeztettek. A Windows XP és a Windows Server 2003 rendszeren a Távsegítség engedélyezheti az RDP protokollt.
-   A tűzfalakra vonatkozó gyakorlati tanácsok, valamint a szabványos alapértelmezett tűzfal-konfigurációk segítséget nyújtanak a hálózatot érő, a szervezet határain kívülről eredő támadások kivédésében. Gyakorlati tanácsként azt javasoljuk, hogy a közvetlen internetkapcsolattal rendelkező számítógépeken a lehető legkevesebb port legyen megnyitva.

#### A Remote Desktop Protocol biztonsági rését (CVE-2012-0173) megkerülő lehetséges megoldások

A lehetséges megoldás olyan beállítást vagy konfigurációs módosítást takar, amely magát a biztonsági rést nem szünteti meg, azonban a frissítés telepítése előtt segít az ismert támadási lehetőségek kivédésében. A Microsoft alaposan megvizsgálta a következő lehetséges megoldásokat, és tájékoztatja a felhasználókat, amennyiben a megoldás csökkenti a működőképességet:

-   **Tiltsa le a Terminálszolgáltatások, a Távoli asztal, a Távsegítség és a Windows Small Business Server 2003 Remote Web Workplace funkciókat, ha már nincs rájuk szüksége**

    Ha már nincs szüksége ezekre a szolgáltatásokra a rendszeren, érdemes őket biztonsági szempontból eltávolítani. A nem használt és felesleges szolgáltatások letiltása csökkenti a biztonsági rések által jelentett veszélyt.

    A Távoli asztal manuális letiltásával kapcsolatban [A Távoli asztal letiltása](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/rdesktop_disable.mspx) című cikkben találhat bővebb információt.

    Ha többet szeretne megtudni arról, miként tilthatja le a Távoli asztalt a csoportházirend segítségével, olvassa el a [Microsoft Tudásbázis 306300. számú cikkét](http://support.microsoft.com/kb/306300).

    A [Távsegítség](http://technet.microsoft.com/en-us/library/bb457177.aspx) weboldalon további tájékoztatást talál a Távsegítség manuális és a csoportházirend segítségével történő letiltásáról.

    A Windows Small Business Server 2003 Terminálszolgáltatások és Remote Web Workplace funkcióinak letiltásáról lásd a TechNet [A Windows Small Business Server 2003 hálózatok biztonsága](http://www.microsoft.com/technet/security/secnews/articles/sec_sbs2003_network.mspx) című cikkét.

-   **Tiltsa le a szervezet külső tűzfalán a 3389-es TCP-portot**

    A rendszer a 3389-es TCP-portot használja az érintett összetevővel való kapcsolat létrehozására. A port külső tűzfalon történő letiltása megakadályozza, hogy a támadók kihasználják a biztonsági rést a tűzfal mögötti rendszereken. Ez segítséget nyújt a hálózatot elérő, a szervezet határain kívülről eredő támadások kivédésében. A szervezet határain lévő portok blokkolása a legjobb módszer az internet alapú támadások elleni védekezésre. A szervezet határain belül azonban a rendszerek biztonsági rése továbbra is fennáll.

    További lehetőségként a Windows XP és a Windows Server 2003 operációs rendszereken a Windows tűzfal segíthet az egyes rendszerek védelmében. Alapértelmezés szerint a Windows tűzfal nem engedélyezi a hozzáférést ehhez a porthoz, kivéve Windows XP Service Pack 2 rendszer esetén, ha a Távoli asztal funkció engedélyezve van. A TechNet [A Távoli asztal tűzfalszabályának engedélyezése és letiltása](http://technet.microsoft.com/en-us/library/cc736451(ws.10).aspx) című cikkében olvashat arról, hogyan tilthatja le a Windows tűzfal Távoli asztalra vonatkozó kivételeit ezeken a platformokon. Ha nem tudja letiltani a Windows tűzfal Távoli asztalra vonatkozó kivételeit, a biztonsági rés hatókörét a helyi hálózatra szűkítheti az összes számítógép (köztük az internetes gépek) alapértékének visszaállításával. Ezáltal csökkentheti az internetről érkező támadások valószínűségét.

    **Megjegyzés** A Windows Small Business Server 2003 rendszer egy Remote Web Workplace nevű funkciót használ. A funkció az RDP-kapcsolatok figyeléséhez a 4125-ös TCP-portot használja. Ha ezt a funkciót használja, meg kell erősítenie, hogy a port a 3389-es porton kívül az interneten is tiltva van.

    **Megjegyzés** Az érintett összetevők manuálisan beállíthatók más port használatára. A műveletek elvégzése után blokkolni kell a további portokat.

-   **A hálózati szintű hitelesítés engedélyezése** **a Windows Vista, Windows 7, Windows Server 2008 és Windows Server 2008 R2 támogatott kiadásain**

    **
    Megjegyzés** A lehetséges megoldás automatikus **Microsoft Fix it** módszerrel történő engedélyezéséhez lásd a [Microsoft Tudásbázis 2685939. számú cikkét](http://support.microsoft.com/kb/2685939).

    A hálózati szintű hitelesítés engedélyezésével megakadályozhatja, hogy hitelesítés nélküli támadók használják ki a biztonsági rést. Bekapcsolt hálózati szintű hitelesítés esetén biztonsági rés kihasználásához a támadónak először a célrendszeren érvényes fiókkal hitelesíteni kell magát a Távoli asztal szolgáltatásokban.

    A hálózati szintű hitelesítés használatához a környezetnek meg kell felelnie a következő követelményeknek:

    -   Az ügyfélszámítógépnek legalább a Távoli asztal kapcsolat 6.0-s verzióját kell használnia.
    -   Az ügyfélszámítógépen a Credential Security Support Provider (CredSSP) protokollt támogató, például Windows 7 vagy Windows Vista operációs rendszernek kell futni.
    -   A távoli asztali munkamenetgazda-kiszolgálón Windows Server 2008 R2 vagy Windows Server 2008 operációs rendszernek kell futnia.

    Adott kapcsolat hálózati szintű hitelesítésének konfigurálásához hajtsa végre a következő lépéseket:

    1.  Nyissa meg a távoli asztali munkamenetgazda-kiszolgálón a Távoli asztali munkamenetgazda konfigurálása alkalmazást. A Távoli asztali munkamenetgazda konfigurálása alkalmazás megnyitásához kattintson a **Start** gombra, mutasson az **Felügyeletieszközök**, majd a **Távoli asztali szolgáltatások** elemre, és kattintson a **Távoli asztali munkamenetgazda konfigurálása** lehetőségre.
    2.  A **Kapcsolatok** részen kattintson jobb gombbal a kapcsolat nevére, majd kattintson a **Tulajdonságok** pontra.
    3.  Jelölje be az **Általános** lapon a **Csak a hálózati szintű hitelesítést alkalmazó Távoli asztalt használó számítógépek kapcsolódásának engedélyezése** jelölőnégyzetet.
        Ha be van jelölve a **Csak a hálózati szintű hitelesítést alkalmazó Távoli asztalt használó számítógépek kapcsolódásának engedélyezése** jelölőnégyzet, mégsem engedélyezett, akkor a **Felhasználóhitelesítés megkövetelése a távoli kapcsolatokhoz a hálózati szintű hitelesítés révén** csoportházirend-beállítást engedélyezték és **alkalmazták a távoli asztali munkamenetgazda-kiszolgálón**.
    4.  Kattintson az **OK** gombra.

    **A megoldás hatása.** A Credential Security Support Provider (CredSSP) protokollt nem támogató ügyfélszámítógépek nem érik el a hálózati szintű hitelesítéssel védett kiszolgálókat. **Megjegyzés** Azok a rendszergazdák, akik olyan vegyes hálózati környezetben alkalmazzák ezt az elkerülő megoldást, ahol a Windows XP Service Pack 3 rendszereknek szükségük van a Távoli asztal funkcióra, a [Microsoft Tudásbázis 951608. számú cikkéből](http://support.microsoft.com/kb/951608) kaphatnak tájékoztatást a CredSSP protokoll Windows XP Service Pack 3 rendszeren történő engedélyezéséről.

    A hálózati szintű hitelesítésről a TechNet [Hálózati szintű hitelesítés konfigurálása Távoli asztali szolgáltatásokkal létesített kapcsolatokhoz](http://technet.microsoft.com/en-us/library/cc732713.aspx) című cikkében talál bővebb információt, többek között a hálózati szintű hitelesítés csoportházirenddel történő engedélyezéséről.

#### Gyakori kérdések a Remote Desktop Protocol biztonsági rése (CVE-2012-0173) kapcsán

**Mire terjed ki a biztonsági rés hatása?**
A biztonsági rés távolról történő kódfuttatást tehet lehetővé.

**Mi okozza a biztonsági rést?**
A biztonsági rés akkor keletkezik, amikor a Remote Desktop Protocol a speciálisan kialakított csomagok sorának feldolgozása eredményeként a memóriában nem megfelelően inicializált vagy törölt objektumhoz fér hozzá.

**Mi a Remote Desktop Protocol (RDP)?**
A Remote Desktop Protocol (RDP) lehetővé teszi, hogy a felhasználók virtuális munkamenetet hozzanak létre asztali számítógépükön. Az RDP lehetővé teszi, hogy a távoli felhasználók a számítógépükön lévő össze adatot és alkalmazást elérjék. További információk a [Remote Desktop Protocol](http://msdn.microsoft.com/en-us/library/aa383015(v=vs.85).aspx) című MSDN cikkben.

**Engedélyezett-e alapértelmezésben a Távoli asztal?**
Nem, a rendszergazdai műveletekhez az RDP alapértelmezésben nincs engedélyezve. Rendszerük védelme érdekében azonban a frissítést azoknak az ügyfeleknek is felajánljuk, akiknél az RDP nem engedélyezett. Az erre vonatkozó konfigurálási beállításról a tudnivalókat lásd a TechNet [A Windows Server 2003 operációs rendszer Távoli asztal felügyeleti célokra szolgáltatásának engedélyezése és konfigurálása](http://support.microsoft.com/kb/814590) című cikkében.

**Mire használhatja a biztonsági rést a támadó?**
A biztonsági rést kihasználó támadó teljes mértékben átveheti a megtámadott rendszer vezérlését. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.

**Hogyan használhatja ki a támadó a** **biztonsági rést?**
A nem hitelesített távoli támadó kihasználhatja ezt a biztonsági rést, ha a célként kiválasztott rendszer részére speciálisan kialakított RDP csomagok sorozatát küldi.

**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**
A biztonsági rés elsősorban a terminálkiszolgálókat veszélyezteti. Azok a rendszerek, amelyeknél az RDP nem engedélyezett, nem veszélyeztettek.

**Mi a frissítés feladata?**
A frissítés a biztonsági rés kiküszöböléséhez módosítja azt az eljárást, amellyel a Remote Desktop Protocol a csomagokat feldolgozza a memóriában.

**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés az említett közlemény kiadásának idején?**
Nem. A Microsoft ezt a célt szolgáló koordinált tájékoztatáson keresztül szerzett tudomást a biztonsági résről.

**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**
Nem. A Microsoft nem kapott olyan információt, hogy a biztonsági közlemény eredeti változatának megjelenéséig a biztonsági rést felhasználták volna nyilvános támadásra.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.

A biztonsági frissítés telepítője az alábbi kapcsolók használatát támogatja:
  
| Kapcsoló                           | Leírás                                                                                                                                                              |  
|------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h,** **/help**              | A támogatott kapcsolóknál megjeleníti a súgót.                                                                                                                      |  
| **/quiet**                         | Nem jeleníti meg az állapot- és hibaüzeneteket.                                                                                                                     |

| **/warnrestart:&lt;seconds&gt;**   | Ha ezt **/quiet** elemmel kapcsolja össze, a telepítő figyelmezteti a felhasználót az újraindítás előtt.                                                            |  
| **/promptrestart**                 | Ha ezt **/quiet** elemmel kapcsolja össze, a telepítő figyelmeztetést küld az újraindítás előtt.                                                                    |  
| **/forcerestart**                  | Ha ezt **/quiet** elemmel kapcsolja össze, a telepítő kényszerített módon bezárja az alkalmazásokat, és újraindítja a rendszert.                                    |

| **/uninstall /kb:&lt;KB-szám&gt;** | Eltávolítja a biztonsági frissítést.                                                                                                                                |
