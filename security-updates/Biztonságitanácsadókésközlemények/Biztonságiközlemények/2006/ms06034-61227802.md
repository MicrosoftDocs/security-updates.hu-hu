A biztonsági rés részletes ismertetése

<span></span>
#### Hibás Active Server Pages objektumot használó Internet Information Services biztonsági rés - CVE-2006-0026

Az Internet Information Services (IIS) biztonsági rése távoli kódfuttatást tesz lehetővé. A támadó egy speciális Active Server Pages (ASP) fájllal használhatja ki a biztonsági rést, így potenciálisan lehetőséget nyit a távoli kódfuttatásra, amennyiben az Internet Information Services (IIS) kezeli a speciálisan létrehozott fájlt. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását.

#### A hibás Active Server Pages ASP elemeket használó Internet Information Services biztonsági rése okozta hibákat enyhítő tényezők - CVE-2006-0026:

-   IIS 5.0 és IIS 5.1 esetén az ASP alapértelmezésben engedélyezett alkalmazásai a „Készletezés folyamatból” alkalmazásban futnak, tehát a DLLHOST.exe által, amely viszont az alacsony jogosultságú IWAM\_&lt;machinename&gt; fiókban fut.
-   Alapértelmezés szerint az IIS 5.1 Windows XP Professional rendszeren és az IIS 6.0 Windows Server 2003 rendszeren nem engedélyezett.
-   Alapértelmezésben az ASP nem engedélyezett az IIS 6.0 alkalmazáshoz. Ha engedélyezi az ASP alkalmazást, az a W3WP.exe munkavégző folyamat környezetében fut alacsony jogosultságú „NetworkService” fiókként.
-   A támadó csak érvényes bejelentkezési adatok birtokában használhatja ki ezt a biztonsági rést. Azonban ha a kiszolgálót úgy konfigurálták, hogy a felhasználóknak - akár névtelenül vagy hitelesítést követően is - lehetővé tegye webes tartalom, pl. ASP-oldalak feltöltését weboldalakra, a biztonsági rés miatt a kiszolgáló támadás áldozatává válhat.

#### Hibás Active Server Pages objektumot használó Internet Information Services biztonsági rés lehetséges megoldásai - CVE-2006-0026

Nincs lehetséges megoldás a biztonsági rés megszüntetésére.

#### Gyakran feltett kérdések a hibás Active Server Pages objektumot használó Internet Information Services biztonsági rés hibáival kapcsolatban - CVE-2006-0026

**Mire terjed ki a biztonsági rés hatása?**
Az Internet Information Services (IIS) szolgáltatásban távoli kódvégrehajtást lehetővé tévő biztonsági rés található, amiatt, ahogyan az IIS az Active Server Pages (ASP) elemeket kezeli. A támadó egy speciális Active Server Pages (ASP) fájllal használhatja ki a biztonsági rést, így potenciálisan lehetőséget nyit a távoli kódfuttatásra, amennyiben az Internet Information Services (IIS) kezeli a létrehozott fájlt. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását.

**Mi okozza a biztonsági rést?**
Ellenőrizetlen puffer az IIS alkalmazásban.

**Mi az Active Server Pages (ASP)?**
A Microsoft Active Server Pages (ASP) kiszolgálóoldali parancsfájlkezelő technológia, amely dinamikus és interaktív webes alkalmazások létrehozására alkalmas. Az ASP oldal olyan HTML oldal, amely kiszolgálóoldali parancsfájlokat tartalmaz, melyeket a webes kiszolgáló kezel, mielőtt elküldené azokat a felhasználó böngészőjére.

**Mire használhatja a biztonsági rést a támadó?**
A biztonsági rést kihasználó támadó teljes mértékben átveheti a megtámadott rendszer vezérlését.

**Kik használhatják ki a biztonsági rést?**
A támadónak a rés kihasználásához érvényes bejelentkezési adatokkal kell rendelkeznie a kiszolgálóra. Azonban ha a kiszolgálót úgy konfigurálták, hogy a felhasználóknak - akár névtelenül vagy hitelesítést követően is - lehetővé tegye webes tartalom, pl. ASP-oldalak feltöltését weboldalakra, a biztonsági rés miatt a kiszolgáló támadás áldozatává válhat.

**Hogyan használhatja ki a támadó a biztonsági rést?**
A támadó megpróbálhatja kihasználni a biztonsági rést, ha speciálisan erre a célra kialakított ASP fájlt hoz létre, és azt feltölti a megtámadni kívánt rendszerre. Ha az IIS kezeltea fájlt, az érintett rendszer végrehajthatja a kódot.

**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**
A biztonsági rés az IIS 5.0 alkalmazást futtató Windows 2000 Service Pack 4 rendszereket érinti a legjobban, mivel az IIS itt alapértelmezésben engedélyezett. A szolgáltatás engedélyezése esetén a Windows XP Professional és a Windows Server 2003 rendszerek is veszélyeztetettek.

**Kihasználható a biztonsági rés az interneten keresztül?**
Igen. A támadó megkísérelheti a biztonsági rést az interneten keresztül kihasználni, amennyiben rendelkezik a kiszolgálón írási jogosultsággal. A tűzfalakra vonatkozó gyakorlati tanácsok, valamint a szabványos alapértelmezett tűzfal-konfigurációk segítséget nyújtanak az internetről érkező támadások kivédésében. A Microsoft közzétett információkat a számítógép védelmét segítő megoldásokról. Végfelhasználók keressék fel [A számítógép védelme webhelyet](http://go.microsoft.com/fwlink/?linkid=21169). Informatikai szakemberek látogassanak el a [Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171) webhelyre.

**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés az említett közlemény kiadásának idején?**
Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről. A Microsoft nem kapott olyan információt, hogy e közlemény eredeti változatának megjelenéséig a biztonsági rést a nyilvánosság tudomására hozták volna.

**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**
Nem. E közlemény eredeti verziójának kiadásáig a Microsoft nem kapott olyan információt, amely arra utalt volna, hogy ezt a biztonsági rést nyilvános támadásra használták, és nem találkozott olyan nyilvánosságra hozott leírásokkal, amelyek tartalmazták volna egy sikeres támadás forgatókönyvét.
