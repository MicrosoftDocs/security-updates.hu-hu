A Microsoft számú biztonsági közleménye MS10-022 - Fontos
=========================================================

A VBScript parancsértelmezőben lévő biztonsági rés távolról történő programkódfuttatást tehet lehetővé (981169)
---------------------------------------------------------------------------------------------------------------

Közzétéve: 2010. április 13.

**Verzió:** 1.0

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Microsoft Windows rendszeren futó VBScript alkalmazás távolról történő kódfuttatást lehetővé tevő, nyilvánosan jelentett biztonsági rését szünteti meg. Ez a biztonsági frissítés Microsoft Windows 2000, Windows XP és Windows Server 2003 esetén Fontos besorolású. Windows Server 2008, Windows Vista, Windows 7 és Windows Server 2008 R2 esetén a fenyegetett kód nem használható ki, de mivel a kód jelen van, a frissítés lényeges biztonsági intézkedésnek minősül, noha nem rendelkezik súlyossági besorolással. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági rés távoli kódfuttatást tesz lehetővé, ha egy rosszindulatú webhelyen speciálisan kialakított párbeszédpanel jelenik meg egy webhelyen és a felhasználó megnyomja az F1 gombot, ekkor ugyanis a Windows Súgó rendszere a támadó által létrehozott Windows alapú súgófájllal indul el. Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a biztonsági rést kihasználó támadó teljes mértékben átveheti az irányítást a számítógép felett,

A biztonsági frissítés a biztonsági rést a VBScript program súgófájlok védett módban történő feldolgozási módjának módosításával szünteti meg. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

A biztonsági frissítés megszünteti az először a [981169. számú Microsoft biztonsági tanácsadóban](http://technet.microsoft.com/security/advisory/981169) ismertetett biztonsági rést.

**Javaslat.**  A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft a minél előbbi telepítést javasolja frissítéskezelő szoftver vagy a Microsoft Update szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** A [Microsoft Tudásbázis 981169. számú cikkében](http://support.microsoft.com/kb/981169) olvasható, hogy a felhasználók a biztonsági frissítés telepítésekor milyen ismert problémákkal találkozhatnak. A cikkben a problémákra javasolt megoldások is szerepelnek.

#### Érintett és nem érintett szoftverek

Az itt felsorolt szoftvereket a verziók és változatok érintettségének felderítése érdekében teszteltük. A többi verzió, illetve kiadás támogatási életciklusa végére ért vagy nem érintett. Az egyes szoftverek és azok verzióinak támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.

Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban
----------------------------------------------------------------

**Milyen verziójú VBScript van telepítve a rendszerre?**  
Az adott rendszer VBScript verzióját az Internet Explorerben szereplő legnagyobb verzió, a Windows operációs rendszer vagy a rendszerre telepített Windows Scripting Host határozza meg, és a Microsoft Internet Explorer, a Windows Service Pack vagy a Windows Scripting Host újabb verziójának telepítésével módosulhat. További információért lásd a [VBScript verzióinformáció](http://msdn.microsoft.com/en-us/library/4y5y7bh5(vs.85).aspx) című MSDN-cikket.

A rendszeren futó VBScript verzió meghatározásához hajtsa végre a következő lépéseket:

1.  Nyissa meg a Windows Intéző alkalmazást.
2.  Navigáljon a **%systemroot%\\system32** könyvtárba.
3.  Kattintson jobb egérgombbal a **vbscript.dll**-re, válassza a **Tulajdonságok** lehetőséget, majd kattintson a **Részletek** fülre.

A verziószám a **Fájlverzió** mezőben jelenik meg. Ha például a fájlverzió számának elején 5.8 áll, például 5.8.7600.16385, akkor a VBScript 5.8 van telepítve a rendszerre.

**VBScript 5.1 verziót használok. Érinti a rendszeremet a probléma?**  
Igen, a KB981350 biztonsági frissítés vonatkozik az Ön rendszerére, és VBScript verzióját VBScript 5.6 verzióra fogja frissíteni. A frissítésről további információért lásd a [Microsoft Tudásbázis 981350. számú cikkét](http://support.microsoft.com/kb/981350).



**Megjegyzés** A VBScript 5.1 verzió csak a Microsoft Windows 2000 Service Pack 4 olyan új telepítéseinél van jelen, melyekhez az Internet Explorer 6 még nem lett telepítve. Ilyen rendszer esetén a biztonsági frissítés telepítése a VBScript verziót VBScript 5.6 verzióra frissíti.

**Windows** **Vista, Windows Server 2008, Windows 7, illetve Windows Server 2008 R2 rendszert használok. Miért kapom meg ezt a frissítést?**  
A biztonsági rés egy Windows fájlban (vbscript.dll) van jelen, mely a Windows Vista, Windows Server 2008, Windows 7 és Windows Server 2008 R2 támogatott kiadásaiban található meg. A Microsoftnak nincsen tudomása olyan eljárásról, mely lehetővé tenné a biztonsági rés kihasználását az ezeket az operációs rendszereket használó számítógépeken. Lényeges biztonsági intézkedésként a biztonsági frissítés elérhető a Windows Vista, Windows Server 2008, Windows 7 és a Windows Server 2008 R2 bármely támogatott kiadását használó felhasználók számára.

**Ha alkalmaztam a 981169. számú tanácsadóban ismertetett megoldásokat, a frissítés telepítése előtt vissza kell vonnom azokat?**  
Ha alkalmazta a [981169. számú Microsoft biztonsági tanácsadóban](http://technet.microsoft.com/security/advisory/981169) ismertetett megoldásokat, a frissítés telepítése előtt nem kell azokat visszavonnia. Ha vissza kívánja vonni a megoldásokat, biztonságosabb, ha ezt a telepítés frissítése után teszi. A tanácsadóban szereplő megoldások megegyeznek az ebben a közleményben javasoltakkal. Ha nem vonja vissza a megoldást, kérjük, a további részletekért tekintse át a vonatkozó információkat **A megoldás hatása** című részben.

**Hol találhatók a fájlinformációs adatok?**

**A biztonsági közleményben tárgyalt szoftvernél korábbi kiadással rendelkezem. Mi a teendő?**

VBScript Súgó gombjához tartozó biztonsági rés - CVE-2010-0483
--------------------------------------------------------------

A távoli kódfuttatás lehetővé teszi, hogy a VBScript alkalmazás együttműködjön a Windows súgófájlokkal az Internet Explorer használata közben. Ha egy rosszindulatú webhelyen speciálisan kialakított párbeszédpanel jelenik meg és a felhasználó megnyomja az F1 gombot, a Windows Súgó rendszere a támadó által létrehozott Windows alapú súgófájllal indul el. Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a biztonsági rést kihasználó támadó teljes mértékben átveheti az irányítást a számítógép felett, Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén A Windows Server 2003 alapú rendszereken az Internet Explorer [fokozott biztonsági beállításai](http://msdn.microsoft.com/en-us/library/ms537180(vs.85).aspx) alapértelmezésben be vannak kapcsolva. Ez enyhíti a biztonsági rés súlyosságát.

A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2010-0483](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0483).

#### A VBScript Súgó gombjához tartozó biztonsági rés hibáit enyhítő tényezők - CVE-2010-0483

Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:

-   A biztonsági rés Windows Vista, Windows Server 2008, Windows 7, illetve Windows Server 2008 R2 rendszeren nem használható ki.
-   A biztonsági rés kihasználásához a felhasználó közreműködése szükséges. A biztonsági rés akkor sem használható ki a felhasználó közreműködése nélkül, ha rosszindulatú webhelyre látogat. A támadónak rá kell vennie a felhasználót, hogy nyomja meg a billentyűzet F1 gombját, miközben a webhelyen egy parancsfájlon alapuló párbeszédpanel van megnyitva.
-   Webalapú támadás esetén a támadónak olyan webhelyet kell üzemeltetnie, amely egy, a biztonsági rés kihasználását lehetővé tevő weblapot tartalmaz. Az érintett weboldalak és a felhasználó által biztosított tartalomnak, esetleg hirdetéseknek helyet adó weboldalak is tartalmazhatnak a biztonsági rést kihasználni képes különlegesen kialakított tartalmat. A támadóknak azonban nem áll módjukban a felhasználókat ezeknek a weboldalaknak a látogatására kényszerítenie. Ehelyett a támadóknak egy adott webhelyre kell csábítaniuk a felhasználókat. Ezt rendszerint úgy érik el, hogy ráveszik őket arra, hogy kattintsanak rá egy e-mailben lévő hivatkozásra vagy azonnali üzenetkezelőben megjelenő kérésre.
-   Alapértelmezés szerint minden támogatott Microsoft Outlook, Microsoft Outlook Express és Windows Mail verzió a Tiltott helyek zónában nyitja meg a HTML e-mail üzeneteket. Ilyenkor a támadó nem képes a biztonsági rés kihasználásával rosszindulatú kódot futtatni. A Tiltott helyek zóna megszüntetheti a biztonsági rés kihasználását megkísérlő támadásokat azáltal, hogy megakadályozza az Active Scripting és az Active X-vezérlők használatát HTML e-mail üzenetek olvasásakor. Ha azonban a felhasználó egy e-mail üzenetben küldött hivatkozásra kattint, a biztonsági rés még mindig fennáll egy webalapú támadáson keresztül. Az Outlook 2007 ezen felül eltérő összetevőn keresztül képezi le a HTML e-maileket, ezzel megszünteti a kihasználás lehetőségét.
-   Az Internet Explorer böngésző Windows Server 2003 rendszeren alapértelmezés szerint korlátozott üzemmódban [fokozott biztonsági beállításokkal](http://go.microsoft.com/fwlink/?linkid=92039)működik. Ez az üzemmód az Internet zóna biztonsági szintjét Magas értékre állítja be. Ez enyhíti az olyan webhelyek hibáit, melyeket nem adtak hozzá az Internet Explorer Megbízható helyek zónájához.
-   A biztonsági rést kihasználó támadó a helyi felhasználóval egyező hozzáférést nyer a rendszerhez. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb lehet az érintettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén.

#### A VBScript Súgó gombjához tartozó biztonsági rés lehetséges megoldásai - CVE-2010-0483

A lehetséges megoldás olyan beállítást vagy konfigurációs módosítást takar, amely magát a biztonsági rést nem szünteti meg, azonban a frissítés telepítése előtt segít az ismert támadási lehetőségek kivédésében. A Microsoft alaposan megvizsgálta a következő lehetséges megoldásokat, és tájékoztatja a felhasználókat, amennyiben a megoldás csökkenti a működőképességet:

-   **Webhelyen megjelenő felhívásra ne nyomja meg az F1 gombot**

    A biztonsági rés sikeres kihasználásához a felhasználónak meg kell nyomnia a billentyűzet F1 gombját. Az elemzés azt mutatja, hogy ha a felhasználók nem nyomják meg az F1 gombot a billentyűzeten, a biztonsági rés kihasználására nincs lehetőség.

    Vagyis a rosszindulatú webhelyeken megpróbálják rávenni a felhasználókat, hogy nyomják meg az F1 gombot. Egy ilyen weboldal párbeszédablakok végtelen sorozatát hívhatja elő, amelyben felszólítják a felhasználót, hogy az ablaksor bezárásához nyomja meg az F1 gombot, vagy információkat (árakat, súgót) ajánlhatnak az F1 gomb révén.

    A felhasználóknak érdemes kerülni az F1 megnyomását webhelyeken vagy más internetes tartalom révén. Ha az F1 gomb megnyomására felszólító párbeszédpanel ismételten megjelenik, jelentkezzen ki a rendszerből, vagy a Feladatkezelővel zárja le az Internet Explorer folyamatot.

-   **Hozzáférés korlátozása a Windows Súgó rendszeréhez**

#### Gyakran feltett kérdések a VBScript Súgó gombjához tartozó biztonsági réssel kapcsolatban - CVE-2010-0483

**Mire terjed ki a biztonsági rés hatása?**  
A biztonsági rés távolról történő kódfuttatást tehet lehetővé. A biztonsági rést kihasználó támadó a bejelentkezett felhasználóval egyező hozzáférést nyer a rendszerhez. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén

**Mi okozza a biztonsági rést?**  
A biztonsági rést az okozza, ahogyan a VBScript együttműködik a Windows súgófájlokkal az Internet Explorer használata során. Noha a Windows súgófájlok eleve nem biztonságos fájlok, a VBScript használata az érintett rendszereken lehetővé teszi, hogy a támadó egy weboldalon megjelenő párbeszédpanelben meghatározza a súgófájl helyét. Ha a látogató egy ilyen párbeszédpanel megjelenítése közben megnyomja az F1 gombot, a speciálisan kialakított súgófájlt a Windows Súgó rendszere letöltheti, ami tetszőleges programkód végrehajtására adhat lehetőséget az aktuálisan bejelentkezett felhasználó biztonsági környezetében.

**Mi a VBScript?**  
A VBScript (Visual Basic Script) egy alkalmazott, objektum alapú parancsnyelv, amelyet gyakran használnak arra, hogy rugalmasabbá és interaktívabbá tegyék a weboldalakat. A VBScript parancsfájlok csak fordító vagy állomás, például Active Server Pages (ASP), Internet Explorer vagy Windows Script Host jelenlétében futtathatók. További információért lásd a [VBScript alapok](http://msdn.microsoft.com/en-us/library/0ad0dkea(vs.85).aspx) című MSDN-cikket.

**Melyek a nem biztonságos fájltípusok?**  
A Microsoft a felhasználók védelme érdekében a „nem biztonságos fájltípusok” fogalmat támadó által fájlokba beágyazható futtatható tartalmak megnevezésére használja. További információért lásd „A Microsoft termékekben futtatható tartalmak bemutatása” című ismertetőt, illetve a [Microsoft Tudástár 291369. számú cikkét](http://support.microsoft.com/kb/291369).

**Mi az F1 billentyű?**  
Az F1 a normál billentyűzeten megtalálható 12 funkcióbillentyű (F1–F12) egyike. Ezek általában a felső billentyűsorban találhatók. A felhasználók a funkciógombok segítségével egyetlen gombnyomással elérhetnek egyes alkalmazási funkciókat. Az F1 gomb a legtöbb alkalmazásban a Súgót hívja elő. Sokan találkoztak már „A súgó megjelenítéséhez nyomja meg az F1 gombot” utasítással.

**Mi a mélyreható biztonsági megoldás?**  
Az adatbiztonság területén a mélységben tagolt védelem olyan struktúrát jelent, melyben egy hálózat vagy egy rendszer biztonságát több védelmi réteggel védik a támadók ellen.

**Mire használhatja a biztonsági rést a támadó?**  
A biztonsági rést sikeresen kihasználó támadó tetszőleges programkódot futtathat a bejelentkezett felhasználó nevében. Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a támadó teljes mértékben átveheti az irányítást a számítógép felett.

**Hogyan használhatja ki a támadó a biztonsági rést?**  
A biztonsági rés akkor használható ki sikeresen, ha a támadó ráveszi a felhasználót, hogy az meglátogasson egy párbeszédpanelt tartalmazó, speciálisan kialakított weblapot, majd ráveszi, hogy nyomja meg a billentyűzeten az F1 gombot, mialatt a párbeszédpanel látható.

**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**  
Elsősorban olyan ügyfél, munkaállomás vagy terminálkiszolgáló rendszerek vannak veszélyben, amelyekben Internet Explorer böngészővel tekintik meg a weblapokat.

**Kisebb-e a kockázat mértéke, ha az Internet Explorer program Windows Server 2003 vagy Windows Server 2008 rendszeren fut?**  
Az Internet Explorer böngésző Windows Server 2003 és Windows Server 2008 rendszeren alapértelmezés szerint korlátozott üzemmódban [fokozott biztonsági beállításokkal](http://go.microsoft.com/fwlink/?linkid=92039) működik, ami csökkenti a biztonsági rés okozta kockázatot. Az Internet Explorer fokozott biztonsági beállításai olyan beépített Internet Explorer beállításokat takarnak, amelyek csökkentik annak lehetőségét, hogy egy felhasználó vagy rendszergazda különlegesen kialakított webes tartalmat töltsön le és futtasson a kiszolgálón. Ez enyhíti az olyan webhelyek hibáit, melyeket nem adtak hozzá az Internet Explorer Megbízható helyek zónájához. Lásd még: [Az Internet Explorer fokozott biztonsági beállításainak kezelése](http://go.microsoft.com/fwlink/?linkid=92055).

**Az Internet biztonsági zóna Magas szintre állítása mekkora védelmet nyújt a biztonsági réssel szemben?**  
Az Internet zóna Magas biztonsági beállítása megvéd a biztonsági rés kihasználása ellen, mivel letiltja az ahhoz szükséges Active Scripting funkciót.

**A biztonsági rést miért nem lehet kihasználni Windows** **Vista, Windows Server 2008, Windows 7, illetve Windows Server 2008 R2 rendszerben?**  
A Windows Vista, Windows Server 2008, Windows 7 és a Windows Server 2008 R2 architekturális fejlesztéseinek köszönhetően az Internet Explorer más módon működik együtt a Windows operációs rendszerrel és a VBScript alkalmazással. Ez megakadályozza a biztonsági rés kihasználását még akkor is, ha a biztonsági rés által fenyegetett kód jelen van azokon a rendszereken, melyekre a Windows Súgó telepítve van.

**Mi a frissítés feladata?**  
A biztonsági frissítés a Windows operációs rendszerek érintett verzióin a biztonsági rést a VBScript program súgófájlok védett módban történő feldolgozási módjának módosításával szünteti meg. Windows Vista, Windows Server 2008, Windows 7 és Windows Server 2008 R2 esetén nem ismert ezt a biztonsági rést kihasználó támadási módszer, de a frissítés lényeges biztonsági intézkedésként telepíthető.

**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**  
Igen. A biztonsági rés nyilvánosságra került, A normál biztonsági rés besorolást és a [CVE-2010-0483](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0483) azonosítót kapta. A biztonsági rést elsőként a [Microsoft 981169. számú biztonsági tanácsadója](http://technet.microsoft.com/security/advisory/981169) ismertette.

**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**  
Nem. A Microsoft nem kapott olyan információt, hogy a biztonsági közlemény eredeti változatának megjelenéséig a biztonsági rést felhasználták volna nyilvános támadásra.

**A frissítés telepítésének ellenőrzése**

-   **Microsoft Baseline Security Analyzer**

    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        
	**Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        
	**Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.

**A frissítés telepítésének ellenőrzése**

-   **Microsoft Baseline Security Analyzer**

    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        
	**Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        
	**Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.
