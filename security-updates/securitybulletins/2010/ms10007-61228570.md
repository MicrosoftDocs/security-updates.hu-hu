A Microsoft számú biztonsági közleménye MS10-007 - Kritikus
===========================================================

A Windows rendszerhéjkezelő biztonsági rése, amely távoli programkódfuttatást tehet lehetővé (975713)
-----------------------------------------------------------------------------------------------------

Közzétéve: 2010. február 9.

**Verzió:** 1.0

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés közvetlenül jelentett biztonsági rést szüntet meg a Microsoft Windows 2000, Windows XP és Windows Server 2003 rendszerben. A frissítés a többi Windows rendszert nem érinti. A biztonsági rés lehetővé teszi programkód távoli végrehajtását, ha valamely alkalmazás, például egy webböngésző, speciálisan kialakított adatokat továbbít a Windows rendszerhéjkezelőn keresztül a ShellExecute API-függvény részére.

A biztonsági frissítés Microsoft Windows 2000, Windows XP és Windows Server 2003 rendszerek összes támogatott kiadásának vonatkozásában ez a frissítés kritikus besorolásúnak minősül. További tudnivalókért tekintse meg az **Érintett és nem érintett szoftverek** c. részt.

A biztonsági frissítés a biztonsági rés elhárítása érdekében javítja azt a módot, ahogyan a ShellExecute API a bemeneti paramétereket érvényesíti. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft az azonnali telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** Nincsenek

URL-hitelesítés biztonsági rése - CVE-2010-0027  
-----------------------------------------------
  
A távoli kódfuttatást lehetővé tévő biztonsági rés a Microsoft Windows érintett verzióiban létezik. A biztonsági rés azért létezik, mert helytelen a ShellExecute API függvénynek küldött bemeneti paraméterek érvényesítése. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2010-0027](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0027).
  
#### Az URL-hitelesítés biztonsági résének (CVE-2010-0027) súlyosságát enyhítő tényezők
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   A biztonsági rést sikeresen kihasználó támadó tetszőleges programkódot futtathat a bejelentkezett felhasználó nevében. Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a támadó teljes mértékben átveheti az irányítást a számítógép felett. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén
  
#### Lehetséges megoldások az URL-hitelesítés biztonsági résének (CVE-2010-0027) kiküszöbölésére
  
A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### Gyakran ismételt kérdések az URL-hitelesítés biztonsági résével (CVE-2010-0027) kapcsolatban
  
**Mire terjed ki a biztonsági rés hatása?**    
A biztonsági rés távolról történő kódfuttatást tehet lehetővé. A biztonsági rést sikeresen kihasználó támadó ezután ugyanolyan jogosultsággal végezhet műveleteket, mint az aktuálisan bejelentkezett felhasználó.
  
**Mi okozza a biztonsági rést?**    
Amikor egy alkalmazás, például egy webböngésző, a ShellExecute API függvényt használja, hogy speciálisan kialakított adatokat dolgozzon fel, a ShellExecute helytelenül érvényesítheti az adatfolyamatot, és a helyi ügyfélrendszerről hajt végre bináris kódot.
  
**Mi a Windows rendszerhéj?**    
A Windows felhasználói felülete az operációs rendszer működtetéséhez és különféle alkalmazások futtatásához szükséges számos objektumhoz biztosít hozzáférést. Ezek közül a legtöbb és legismertebb objektumok a számítógép lemezmeghajtóin tárolt mappák és fájlok. A számítógép adathordozóin olyan virtuális objektumok is találhatók, amelyek bizonyos feladatok végrehajtását teszik lehetővé. Ilyen feladat például a fájlok távoli nyomtatóra küldése vagy a hozzáférés a Lomtárhoz. A rendszerhéj az objektumokat hierarchikus névtérbe rendezi, így a felhasználók számára egységes és hatékony hozzáférést, valamint kezelési lehetőséget biztosít.
  
**Mi a ShellExecute?**    
A ShellExecute a Windows rendszerhéj alkalmazásprogramozói felületét (API) képező függvények része. Műveletet hajt végre egy megadott fájlon. Ez lehet például az adott fájl típusának megfelelő helyes kezelő meghívása. A ShellExecute függvényről további tájékoztatást ad az MSDN [ShellExecute Function](http://msdn.microsoft.com/en-us/library/bb762153(vs.85).aspx) című cikke.
  
**Mi az alkalmazásprogramozási felület (API)?**    
Az API olyan függvények készlete, amelyek valamely összetevővel, alkalmazással vagy az operációs rendszerrel végzett munkához használhatók. Az API jellemzően valamely konkrét működést biztosító egy vagy több DLL fájlból áll.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó tetszőleges programkódot futtathat a bejelentkezett felhasználó nevében. Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a támadó teljes mértékben átveheti az irányítást a számítógép felett. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A támadó megadhat egy olyan tartalmat mint egy speciálisan kialakított weboldal, aminek eredményeként az alkalmazás rosszindulatú karakterláncot elemez. Amikor ez a tartalom egy ShellExecute API híváshoz kerül, például hiperhivatkozásként, akkor lehetővé válhat, hogy a támadó a bejelentkezett felhasználó tudta nélkül bináris kódot hajtson végre az ügyfélgépről.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
A bejelentkezett felhasználó környezetét kihasználó biztonsági rés által elsősorban azok az ügyfélrendszerek veszélyeztettek, amelyek nem megbízható internetes tartalmat böngésznek.
  
**Mi a frissítés feladata?**    
Ez a frissítés úgy kezeli a biztonsági rést, hogy kijavítja azt a módot, ahogyan a ShellExecute API érvényesíti a bemeneti paramétereket a Microsoft Windows 2000, a Windows XP és Windows Server 2003 támogatott kiadásain.
