A Microsoft számú biztonsági közleménye MS09-059 - Fontos
=========================================================

Az LSASS biztonsági rése szolgáltatásmegtagadást okozhat (975467)
-----------------------------------------------------------------

Közzétéve: 2009. október 13.

**Verzió:** 1.0

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Microsoft Windows közvetlenül jelentett biztonsági rését szünteti meg. A biztonsági rés szolgáltatásmegtagadást okozhat, ha a támadó rosszindulatú céllal létrehozott csomagot küld az NTLM hitelesítési folyamat során.

Ez a fontos besorolású biztonsági frissítés a Windows XP, Windows Server 2003, a Windows Vista, Windows 7 és a Windows Server 2008 összes támogatott kiadását érinti. A Windows XP és a Windows Server 2003 támogatott verziót csak akkor érinti a biztonsági frissítés, ha a felhasználó előzőleg már telepítette a KB968389 közleményben kiadott, nem biztonsági jellegű frissítést. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági frissítés a biztonsági rés megszüntetése érdekében a hitelesítési folyamat során használt értékkészletekhez további érvényesítési eljárásokat alkalmaz. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.**  A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft a minél előbbi telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** [A Microsoft Tudásbázis 975467](http://support.microsoft.com/kb/975467) cikkében olvasható, hogy a felhasználók a biztonsági frissítés telepítésekor milyen ismert problémákkal találkozhatnak. A cikkben a problémákra javasolt megoldások is szerepelnek.

Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban
----------------------------------------------------------------

**Milyen rendszereket érint a biztonsági rés?**  
A biztonsági rés a Windows Vista, Windows Server 2008, Windows 7 és Windows Server 2008 R2 rendszert érinti. Érinti továbbá a Windows XP és a Windows Server 2003 rendszert is, amennyiben KB968389 számú, nem biztonsági frissítés, amely a javított védelmet alkalmazza, telepítve van a rendszeren.

**Érinti a probléma a számítógépemet, ha nem telepítettem a KB968389 frissítést?**  
A Windows XP és a Windows Server 2003 támogatott verzió a KB968389 telepítésének hiányában nem fenyegetettek. A Windows Vista, Windows Server 2008, Windows 7 és a Windows Server 2008 R2 azon példányai, amelyeken a KB968389 frissítés nincs telepítve, fenyegetettek.

**A rendszer miért csak a KB968389 frissítést ajánlja fel a Windows Update vagy a Microsoft Update szolgáltatáson keresztül?**  
Ha a KB968389 frissítést a Windows Update vagy a Microsoft Update szolgáltatáson keresztül szerzi be, a jelen frissítés beszerzését követően, a rendszer automatikusan telepíti a KB975467 frissítést. A KB968389 sikeres telepítését követően mind a KB968389 és a MS09-059, KB975467 biztonsági frissítés telepített szoftverként szerepel majd a rendszeren.

Amennyiben a KB968389 frissítést jelen biztonsági frissítés megjelenése előtt már telepítette, a Windows Update és a Microsoft Update szolgáltatás a biztonsági rés megszüntetéséhez a KB975467 biztonsági frissítés ajánlja majd fel.

**Az MBSA támogatja a biztonsági frissítés észlelését Windows 7 és Windows Server 2008 R2 rendszerben?**  
Az MBSA a 2.1.1 verziótól kezdve támogatja a biztonsági frissítés észlelését Windows 7 és Windows Server 2008 R2 rendszerben, az említett operációs rendszerek általános elérhetőségi dátumától kezdve. További információkért lásd: [MBSA](http://www.microsoft.com/mbsa).

**Hol találhatók a fájlinformációs adatok?**  
A fájlinformációs adatokat lásd a **Biztonsági frissítés telepítése** rész hivatkozási táblázataiban.

**A biztonsági közleményben tárgyalt szoftvernél korábbi kiadással rendelkezem. Mi a teendő?**  
Teszteléssel állapították meg a közleményben tárgyalt szoftverkiadások érintettségét. A többi kiadás támogatási életciklusa végére ért. Az egyes szoftverkiadások támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.

A szoftver korábbi kiadásai felhasználóinak számára azt javasoljuk, hogy a potenciális biztonsági veszélyek elkerülése érdekében sürgősen térjenek át valamelyik támogatott kiadású szoftverre. Ha többet szeretne tudni a Windows termékek életciklusáról, keresse fel a [Microsoft támogatási életciklusokkal foglalkozó webhelyét](http://go.microsoft.com/fwlink/?linkid=21742). Ha többet szeretne tudni az ilyen szoftververziók illetve kiadások biztonsági frissítéseire vonatkozó meghosszabbított terméktámogatási időszakokról, keresse fel a [Microsoft terméktámogatási szolgálatát](http://go.microsoft.com/fwlink/?linkid=33328).

Azok az ügyfelek, akik további támogatást igényelnek a korábbi szoftverkiadásokhoz, a támogatási lehetőségekről érdeklődjenek a Microsoft ügyfélkapcsolati képviselőjénél, a műszaki kapcsolattartó menedzsernél vagy a Microsoft megfelelő partnerképviseleténél. Alliance, Premier vagy Authorized szerződéssel rendelkező felhasználóink forduljanak a Microsoft helyi értékesítési irodájához. Az elérhetőségeket keresse a [Microsoft Worldwide Information webhelyen](http://go.microsoft.com/fwlink/?linkid=33329): válasszon egy országot a legördülő listából, majd az **Indítás** gombra kattintva jelenítse meg a telefonszámokat. Telefonáláskor keresse a helyi értékesítési menedzsert. További információ: [Windows operációs rendszerek terméktámogatási időszakával kapcsolatos, gyakran ismételt kérdések](http://go.microsoft.com/fwlink/?linkid=33330).

A Local Security Authority Subsystem Service egészszám-túlcsordulási biztonsági rése - CVE-2009-2524  
----------------------------------------------------------------------------------------------------
  
A Microsoft Windows Local Security Authority Subsystem Service (LSASS) jogok kiterjesztését lehetővé tévő biztonsági rés található, amelyet az NTLM hitelesítés hibás kialakított csomagokat érintő nem megfelelő kezelése okoz. A támadó speciálisan kialakított, névtelen NTLM-hitelesítési kérelmeket hozhat létre, amelyek az LSASS-szolgáltatás összeomlását okozhatják, majd ezt követően újraindítják a számítógépet.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2009-2524](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2524).
  
#### A Local Security Authority Subsystem Service egészszám-túlcsordulási biztonsági résének (CVE-2009-2524) súlyosságát enyhítő tényezők
  
A Microsoft egyelőre nem talált enyhítő tényezőket a biztonsági rés megszüntetésére.
  
#### Lehetséges megoldások a Local Security Authority Subsystem Service egészszám-túlcsordulási biztonsági résének (CVE-2009-2524) kiküszöbölésére
  
A lehetséges megoldás olyan beállítást vagy konfigurációs módosítást takar, amely magát a biztonsági rést nem szünteti meg, azonban a frissítés telepítése előtt segít az ismert támadási lehetőségek kivédésében. A Microsoft alaposan megvizsgálta a következő lehetséges megoldásokat, és tájékoztatja a felhasználókat, amennyiben a megoldás csökkenti a működőképességet:
  
-   **Távolítsa el a KB968389 frissítést a Windows XP és a Windows Server 2003 rendszert működtető számítógépekről**

#### Gyakran feltett kérdések a Local Security Authority Subsystem Service egészszám-túlcsordulási biztonsági résével (CVE-2009-2524) kapcsolatban
  
**Mire terjed ki a biztonsági rés hatása?**    
Ez a biztonsági rés szolgáltatásmegtagadást okoz. Egy különlegesen kialakított hitelesítési csomag lehetővé teheti, hogy a hitelesítést el nem végző támadó a kiszolgálóoldali LSASS-folyamat válaszadását leállíthatja, majd újraindíthatja a számítógépet.
  
**Mi okozza a biztonsági rést?**    
A biztonsági rés azért jelent meg, mert a Windows NTLM implementációja az LSASS szolgáltatásban a hitelesítési folyamat során nem megfelelően kezel bizonyos hibásan kialakított csomagokat. A különlegesen kialakított csomag az LSASS-folyamatban egészszám-alulcsordulást okozhat.
  
**Az LSASS meghatározása**    
Az LSASS szolgáltatás felületet nyújt a helyi biztonság, a tartományi hitelesítés és az Active Directory folyamatainak kezeléséhez, valamint kezeli az ügyfél és a kiszolgáló hitelesítéseit. Az LSASS része továbbá néhány, az Active Directory segédprogramjait támogató szolgáltatás is.
  
**Mi az az NTLM?**    
A Windows Challenge/Response (NT Lan Manager vagy NTLM) hitelesítő protokollt használják azokon a hálózatokon, amelyek Windows operációs rendszereket tartalmaznak, továbbá ugyanez a protokoll használatos különálló rendszereken is. Az NTLM-hitelesítő adatok az interaktív bejelentkezési folyamat során beszerzett adatokon alapszanak, és tartománynevet, felhasználónevet, valamint a felhasználó egyutas hash-elemét tartalmazzák. Az NTLM titkosított kérdés/válasz protokollt használ a kiszolgáló hitelesítéséhez, anélkül, hogy a felhasználó jelszavát a hálózaton keresztül elküldené. A jelszó küldése helyett a hitelesítést kérő rendszernek egy számítást kell elvégeznie, amely bizonyítja, hogy hozzáféréssel rendelkezik a titkosított NTLM-hitelesítő adatokhoz.
  
A hálózaton keresztüli interaktív NTLM-hitelesítés jellemzően két rendszert vesz igénybe: az ügyfélrendszert, amelyen a felhasználó a hitelesítést kérelmezi, valamint a tartományvezérlőt, ahol a felhasználói jelszóval kapcsolatos adatok tárolása történik. A nem interaktív hitelesítés során, amelyre akkor lehet szükség, ha egy már bejelentkezett felhasználónak kell hozzáférést engedélyezni valamely erőforráshoz, pl. kiszolgálóalkalmazáshoz, három rendszer vesz részt a folyamatban: az ügyfél, a kiszolgáló és a tartományvezérlő, amely a kiszolgáló részéről hajtja végre a hitelesítéshez szükséges számításokat.
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó az érintett rendszer automatikus újraindulását idézheti elő.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A támadó a biztonsági rés kihasználásához az NTLM-hitelesítési folyamat során hibásan kialakított csomagokat küldhet.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
Az NTLM-hitelesítést engedélyező kiszolgálókra.
  
**Mi a frissítés feladata?**    
A frissítés biztosítja, hogy az LSASS a hitelesítési folyamat érintet értékeit megfelelően kezelje.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**    
Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**    
Nem. E közlemény eredeti verziójának kiadásáig a Microsoft nem kapott olyan információt, amely arra utalt volna, hogy ezt a biztonsági rést nyilvános támadásra használták, és nem találkozott olyan nyilvánosságra hozott leírásokkal, amelyek tartalmazták volna egy sikeres támadás forgatókönyvét.

    1.  Kattintson a **Start** menü **Keresés** parancsára.

    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.

Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**
