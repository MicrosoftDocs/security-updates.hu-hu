A Microsoft számú biztonsági közleménye MS09-051 - Kritikus
===========================================================

A Windows Media Runtime biztonsági rései távoli programkódfuttatást tesznek lehetővé (975682)
---------------------------------------------------------------------------------------------

Közzétéve: 2009. október 13. | Frissítve: 2009. november 10.

**Verzió:** 2.0

### Általános tudnivalók

#### Összefoglalás

Ez a biztonsági frissítés a Windows Media Runtime két közvetlenül jelzett biztonsági rését szünteti meg. A biztonsági rés távoli kódfuttatást tehet lehetővé, ha a felhasználó megnyit egy különlegesen kialakított médiafájlt, vagy különlegesen kialakított tartalom-adatfolyamot fogad egy webhelyről, illetve bármely webes tartalmat szállító alkalmazásból. A biztonsági réseket kihasználó támadó a helyi felhasználóval egyező hozzáférést nyer a rendszerhez. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén

Ez a biztonsági frissítés kritikus besorolású a DirectShow WMA beszédhangkodek, Windows Media audiodekódoló és hangtömörítés-kezelő tekintetében, a következő operációs rendszerek minden támogatott verziója esetén: Microsoft Windows 2000; Windows XP; Windows Server 2003, az Itanium alapú kiadások kivételével; Windows Vista; és Windows Server 2008, az Itanium alapú kiadások kivételével. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági frissítés azáltal oldja meg a biztonsági rések okozta problémákat, hogy módosítja azt a módot, ahogyan a Windows Media Runtime feldolgozza az ASF-fájlokat és inicializálja a funkciókat a tömörített hangfájlokban. A biztonsági résekkel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft az azonnali telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

**Ismert problémák** Nincsenek

Gyakran ismételt kérdések a biztonsági frissítéssel kapcsolatban  
----------------------------------------------------------------
  
**Miért adták ki újra a közleményt 2009. november 10-én?**    
A Microsoft a frissített közleményben tájékoztat a Microsoft Office 2000 Service Pack 4 rendszeren futó hangtömörítés-kezelő érzékelési problémáját javító frissítés újrakiadásáról. Ez csak észlelési módosítás; a bináris értékek nem változtak. A rendszerüket sikeresen frissített felhasználóknak nem kell újra telepíteniük a frissítést.
  
**Hol találhatók a fájlinformációs adatok?**    
A fájlinformációs adatokat lásd a **Biztonsági frissítés telepítése** rész hivatkozási táblázataiban.
  
**Hogyan kapcsolódik ez a közlemény az MS09-052 közleményhez: A Windows Media Player biztonsági rése, amely távoli programkódfuttatást tesz lehetővé?**    
Ez a közlemény nem kapcsolódik az [MS09-052](http://go.microsoft.com/fwlink/?linkid=163913) közleményhez (A Windows Media Player biztonsági rése, amely távoli programkódfuttatást tesz lehetővé). A teljes védettség biztosításához a felhasználóknak mindkét frissítést telepíteniük kell.
  
**Telepítettem a Windows Media Player alkalmazást a rendszeremre. Miért nem ajánlják fel nekem valamelyik frissítést?**  
Csak akkor kínáljuk fel a rendszerek számára automatikusan ezeket a frissítéseket, ha telepítették a Windows Media Player szoftvert, és a biztonsági rések által érintett, választható audiokodekeket. A KB969878 közleményhez kapcsolódó kodekek csak 32 bites rendszerekben használatosak. A KB954155 közleményhez kapcsolódó kodekek alapértelmezetten nincsenek telepítve az x64-alapú Windows Media Format SDK készlethez, de automatikusan letölthetők a Windows Media Player útján, vagy manuálisan a [Microsoft letöltőközpontjából](http://www.microsoft.com/downloads/en/default.aspx). Ha nincsenek telepítve a kodekek a számítógépen és megkísérli a frissítések manuális telepítését, a következő hibaüzenetek egyikét kaphatja:
  
-   Ennek a frissítésnek a telepítéséhez szükség van a Windows Media Player 9 szoftverre.  
-   Ennek a frissítésnek a telepítéséhez a gépen ott kell lennie a Windows Media Format 9.5 SDK szoftvernek.
  
Ha megkapja ezt az üzenetet, akkor ez a frissítés az Ön rendszerére nem vonatkozik.  Ezek az üzenetek csak akkor jelennek meg, ha megkísérli manuálisan telepíteni a frissítést.
  
**Miért tartalmazza több bejelentett biztonsági rés javítását is ez a frissítés?**    
A frissítés több biztonsági rés javítását is tartalmazza, mivel a problémák megoldásához szükséges módosítások egymáshoz kapcsolódó fájlokban találhatók. A különálló, de egymással majdnem pontosan megegyező frissítések telepítése helyett így a vásárlóknak elegendő ezt az egy frissítést telepíteni.
  
**Mi az a Windows Media Format szoftverfejlesztői készlet (SDK)?**    
A Windows Media Format SDK alkalmazásokat biztosít a fejlesztők számára, a Windows Media Format összetevőinek elérési lehetőségével. A Windows Media Format SDK elsődleges célja annak lehetővé tétele, hogy a fejlesztők létrehozhassanak alkalmazásokat az Advanced Systems Format (ASF) fájlok és hálózati adatfolyamok lejátszására, megírására, szerkesztésére, titkosítására és kézbesítésére. Ezek a fájlok és adatfolyamok általában tartalmaznak a Windows Media audio- és videokodekek használatával kódolt hangi és képi tartalmakat. Az, ASF azonban bármilyen típusú adatot tartalmazhat. További információkért lásd: [Windows Media Format SDK technológiai lap](http://msdn.microsoft.com/en-us/windows/bb190326.aspx).
  
**A biztonsági közleményben tárgyalt szoftvernél korábbi kiadással rendelkezem. Mi a teendő?**    
Teszteléssel állapították meg a közleményben tárgyalt szoftverkiadások érintettségét. A többi kiadás támogatási életciklusa végére ért. Az egyes szoftverkiadások támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.
  
A szoftver korábbi kiadásai felhasználóinak számára azt javasoljuk, hogy a potenciális biztonsági veszélyek elkerülése érdekében sürgősen térjenek át valamelyik támogatott kiadású szoftverre. Ha többet szeretne tudni a Windows termékek életciklusáról, keresse fel a [Microsoft támogatási életciklusokkal foglalkozó webhelyét](http://go.microsoft.com/fwlink/?linkid=21742). Ha többet szeretne tudni az ilyen szoftververziók illetve kiadások biztonsági frissítéseire vonatkozó meghosszabbított terméktámogatási időszakokról, keresse fel a [Microsoft terméktámogatási szolgálatát](http://go.microsoft.com/fwlink/?linkid=33328).
  
Azok az ügyfelek, akik további támogatást igényelnek a korábbi szoftverkiadásokhoz, a támogatási lehetőségekről érdeklődjenek a Microsoft ügyfélkapcsolati képviselőjénél, a műszaki kapcsolattartó menedzsernél vagy a Microsoft megfelelő partnerképviseleténél. Alliance, Premier vagy Authorized szerződéssel rendelkező felhasználóink forduljanak a Microsoft helyi értékesítési irodájához. Az elérhetőségeket keresse a [Microsoft Worldwide Information webhelyen](http://go.microsoft.com/fwlink/?linkid=33329): válasszon egy országot a legördülő listából, majd az **Indítás** gombra kattintva jelenítse meg a telefonszámokat. Telefonáláskor keresse a helyi értékesítési menedzsert. További információ: [Windows operációs rendszerek terméktámogatási időszakával kapcsolatos, gyakran ismételt kérdések](http://go.microsoft.com/fwlink/?linkid=33330).

Windows Media Runtime hangi alkalmazás mintavételi arányát érintő biztonsági rés - CVE-2009-0555  
------------------------------------------------------------------------------------------------
  
A Windows Media Player alkalmazásban az ASF-fájlok kezelési módja miatt távoli kódvégrehajtást lehetővé tevő biztonsági rés jelentkezett. A támadó ennek kihasználásához speciális audiofájlt hoz létre, amely akkor ad lehetőséget távoli kódfuttatásra, amikor a Windows Media Player érintett verziójával lejátsszák. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2009-0555](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0555).
  
#### Windows Media Runtime hangi alkalmazás mintavételi arányát érintő biztonsági rés enyhítő tényezői - CVE-2009-0555
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   Webalapú támadás esetén a támadónak olyan webhelyet kell üzemeltetnie, amely egy, a biztonsági rés kihasználását lehetővé tevő weblapot tartalmaz. Az érintett weboldalak és a felhasználó által biztosított tartalomnak, esetleg hirdetéseknek helyet adó weboldalak is tartalmazhatnak a biztonsági rést kihasználni képes különlegesen kialakított tartalmat. A támadóknak azonban nem áll módjukban a felhasználókat ezeknek a weboldalaknak a látogatására kényszerítenie. Ehelyett a támadóknak az adott webhelyre kell csábítaniuk a felhasználókat. Ezt rendszerint úgy érik el, hogy ráveszik őket arra, hogy kattintsanak rá egy e-mailben lévő hivatkozásra vagy azonnali üzenetkezelőben megjelenő kérésre.  
-   A biztonsági rést kihasználó támadó a helyi felhasználóval egyező hozzáférést nyer a rendszerhez. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb lehet az érintettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén.  
-   A Windows Server 2008 rendszert csak akkor érinti a probléma, ha telepíti a választható felhasználói felület összetevőt is.
  
#### Lehetséges megoldások a Windows Media Runtime hangi alkalmazás mintavételi arányát érintő biztonsági rés esetén - CVE-2009-0555
  
A lehetséges megoldás olyan beállítást vagy konfigurációs módosítást takar, amely magát a biztonsági rést nem szünteti meg, azonban a frissítés telepítése előtt segít az ismert támadási lehetőségek kivédésében. A Microsoft alaposan megvizsgálta a következő lehetséges megoldásokat, és tájékoztatja a felhasználókat, amennyiben a megoldás csökkenti a működőképességet:
  
-   **A wmspdmod.dll regisztráció törlése**
  
    A DLL elem regisztrálása törléséhez futtassa a következő parancsot az emelt szintű parancssorból.
  
    32 bites Windows-rendszer esetén:
  
    `Regsvr32.exe –u %WINDIR%\system32\wmspdmod.dll`
  
    x64 alapú Windows rendszereknél:
  
    `Regsvr32.exe –u %WINDIR%\syswow64\wmspdmod.dll`
  
    **A megoldás hatása:** Az ASF-fájlokban lévő, a Windows Media beszédkodeket használó hangadatfolyamok lejátszása sikertelen.
  
    **A megoldás visszavonása.**
  
    Írja a következő parancsot az emelt szintű parancssorba.
  
    32 bites Windows-rendszer esetén:
  
    `Regsvr32.exe %WINDIR%\system32\wmspdmod.dll`
  
    x64 alapú Windows rendszereknél:
  
    `Regsvr32.exe %WINDIR%\syswow64\wmspdmod.dll`
  
#### GYIK a lehetséges megoldásokról a Windows Media Runtime hangi alkalmazás mintavételi arányát érintő biztonsági rés esetén - CVE-2009-0555
  
**Mire terjed ki a biztonsági rés hatása?**    
A biztonsági rés távolról történő kódfuttatást tehet lehetővé. Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a biztonsági rést kihasználó támadó teljes mértékben átveheti az irányítást a számítógép felett, Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén
  
**Mi okozza a biztonsági rést?**    
A Windows Media Runtime nem megfelelően kezeli a speciálisan kialakított ASF-fájlokat, amelyek a Window Media beszédkodeket igénybe veszik.
  
**A Windows Media Format Runtime bemutatás?**    
A Microsoft Windows Media Format Runtime a Windows Media tartalmat használó alkalmazások számára nyújt adatokat és eszközöket.
  
**Mi a fejlett rendszerformátum (ASF)?**    
Az ASF (Advanced Streaming Format) audio- és videoadatokat tároló fájlformátum, különleges kialakítása miatt kifejezetten alkalmas hálózaton keresztül, pl. interneten való futtatásra. Tömörített formátum, amely adatfolyam-formátumban tartalmaz hang- és videoanyagokat, diavetítéseket és szinkronizált eseményeket. Az ASF lehetővé teszi, hogy a tartalom adatfolyamként jusson el a felhasználóhoz. Az ASF fájlok ASF, WMV vagy WMA kiterjesztéssel rendelkezhetnek.
  
Az ASF a hangfájl háttérformátumára vonatkozik. Az ASF-fájloknak sok különféle fájlkiterjesztése lehet, ezek közül a leggyakoribb a WMA és a WMV. Az ASF-fájlok kézbesíthetők ezenkívül más fájlkiterjesztésekkel is, pl. MP3, ami nem tükrözi a valódi fájlformátumot. A Windows Media Player elemzi a tényleges fájlformátumot, és ennek ellenére megkísérli lejátszani a fájlt.
  
**A Windows Media audiodekódoló bemutatása**    
A Windows Media audiodekódoló dekódolja az adatfolyamokat, amelyeket kódoltak a [Windows Media audiokódoló](http://msdn.microsoft.com/en-us/library/dd756600(vs.85).aspx)használatával. A Windows Media beszédhangkodek a beszédet tartalmazó hangfájlokra optimalizált alacsony bitsebességű tömörítésről gondoskodik. A kodek azért képes ilyen kicsi mintákat előállítani, mert az emberi beszédhangok frekvenciatartománya korlátozott. Ez az optimalizálás azt jelenti, hogy egy dedikált hangkódoló létrehoz alacsony minőségű kimeneteket olyan tartalmakhoz, amelyek bonyolultabb hangokat, például zenét tartalmaznak. A Windows Media beszédhangkodek azonban kompenzálja ezt a lehetséges minőségi problémát azáltal, hogy külön módokat biztosít a beszédhanghoz, a zenéhez és a vegyes tartalomhoz. A kodek elemzi a vegyes tartalmat annak megállapítására, hogy melyik módot kell használni a fájl egyes részeihez. A Windows Media audiodekóderrel kapcsolatos további információkért lásd az MSDN cikket: [A Windows Media beszédhangkodek használata](http://msdn.microsoft.com/en-us/library/dd443456(vs.85).aspx).
  
**Mire használhatja a biztonsági rést a támadó?**    
A biztonsági rést sikeresen kihasználó támadó mesterséges kódot futtathat az érintett rendszeren. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A biztonsági rés akkor használható ki, ha a felhasználó különlegesen kialakított ASF-fájlt nyit meg a Windows Media Player érintett verziójával.
  
Internetes támadás esetén a támadónak olyan webhelyet kell üzemeltetnie, amely a biztonsági rés kihasználását lehetővé tevő, erre a célra kialakított ASF-fájlt alkalmaz. A támadónak nincs lehetősége arra, hogy a felhasználót a különlegesen kialakított webhely megnyitására kényszerítse. Ehelyett a támadóknak az adott webhelyre kell csábítaniuk a felhasználókat. Ezt rendszerint úgy érik el, hogy ráveszik őket arra, hogy kattintsanak rá egy e-mailben lévő hivatkozásra vagy azonnali üzenetkezelőben megjelenő kérésre.
  
E-mailes támadás esetén a támadó erre a célra kialakított ASF-fájl beágyazásával készült médiafájlt küld a biztonsági rés kihasználása céljából, és ráveszi a felhasználót, hogy nyissa meg a fájlt.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
A Windows Media Runtime alkalmazást használó rendszerek (munkaállomások vagy terminálkiszolgálók) vannak a leginkább kitéve a biztonsági rés okozta kockázatnak.
  
**Mi a frissítés feladata?**    
A frissítés a biztonsági rés elhárítása érdekében módosítja azt a módot, ahogyan a Windows Media Runtime az ASF-fájlokat feldolgozza.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**    
Igen. A biztonsági rés nyilvánosságra került, A normál biztonsági rés besorolást és a [CVE-2009-0555](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0555) azonosítót kapta.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**    
Igen. A Microsoft értesült a biztonsági rés kihasználására törekvő, korlátozott mértékű támadásokról.

    Windows 2000, Windows XP és Windows Server 2003 rendszeren:
  
    32 bites Windows-rendszer esetén:
  
    `Echo y| cacls %WINDIR%\system32\msaud32.acm /E /P everyone:N`

    ```
    Takeown.exe /f "%windir%\system32\msaud32.acm"  
    Icacls.exe "%windir%\system32\msaud32.acm " /save %TEMP%\msaud32_acm.TXTI  
    cacls.exe "%windir%\system32\msaud32.acm " /deny everyone:(F)
    ```

    ```
    Takeown.exe /f "%windir%\syswow64\msaud32.acm"

    Icacls.exe "%windir%\syswow64\msaud32.acm " /deny everyone:(F)
    ```
  
    **A megoldás hatása:** A hangtömörítés-kezelő (ACM) elemre támaszkodó alkalmazások nem tudják megfelelően leképezni a tartalmat
  
    **A megoldás visszavonása**
  
    Windows 2000, Windows XP és Windows Server 2003 rendszeren:
  
    32 bites Windows-rendszer esetén:
  
    `cacls “%WINDIR%\system32\msaud32.acm” /E /R everyone`
  
    64 bites Windows-rendszer esetén:
  
    `cacls “%WINDIR%\syswow64\msaud32.acm” /E /R everyone`

    Rendszergazda jogkörrel írja a következő parancsokat a parancssorba:
  
    32 bites Windows-rendszer esetén:
  
    `icacls %windir%\syswow64 /restore %TEMP%\msaud32_acm.TXT`
  
    64 bites Windows-rendszer esetén:
  
    `icacls %windir%\syswow64 /restore %TEMP%\msaud32_acm.TXT`

A Microsoft Windows Media Runtime nem tud megfelelően inicializálni bizonyos funkciókat a tömörített hangfájlokban.
  
**A Windows Media Format Runtime bemutatás?**    
A Microsoft Windows Media Format Runtime a Windows Media tartalmat használó alkalmazások számára nyújt adatokat és eszközöket.
  
**A hangtömörítés-kezelő bemutatása**    
A hangtömörítés-kezelő (ACM) rendszerszintű támogatást nyújt az átlátható futásidejű hangtömörítéshez és kibontáshoz, valamint a hullámforma hangadatok feldolgozásához. Az ACM létező illesztőfelülete hurkokat használ fel az alapértelmezett megfeleltetési algoritmus felülbírálására a hullámforma hangeszközök esetén. Ez lehetővé teszi, hogy az ACM elfogja az eszközmegnyitási hívásokat. A hívás elfogása után az ACM sokféle feladatot végrehajthat a hangadatok feldolgozása érdekében, például beilleszthet külső tömörítőt vagy kibontót a sorozatba. Az ACM-mel kapcsolatos további információkért lásd [A hangtömörítés-kezelő bemutatása](http://msdn.microsoft.com/en-us/library/dd742878(vs.85).aspx) c. MSDN-cikket.
  
**Mire használhatja a biztonsági rést a támadó?**    
Ha valamely felhasználó felügyeleti jogosultságokkal bejelentkezett a rendszerbe, akkor a biztonsági rést kihasználó támadó teljes mértékben átveheti az irányítást a számítógép felett, Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat. A kevesebb jogosultsággal bíró fiókkal rendelkező felhasználók esetében kisebb a veszélyeztetettség, mint a rendszergazdai jogosultságokkal rendelkezők esetén
  
**Hogyan használhatja ki a támadó a biztonsági rést?**    
A biztonsági rés kihasználásához a felhasználónak meg kell nyitnia egy különlegesen kialakított médiafájlt, vagy különlegesen kialakított tartalom-adatfolyamot kell megnéznie egy webhelyen, illetve webes tartalmat szállító alkalmazásban.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**    
A Windows Media Player alkalmazást használó rendszerek (munkaállomások vagy terminálkiszolgálók) vannak a leginkább kitéve a biztonsági rés okozta kockázatnak.
  
**Mi a frissítés feladata?**    
A frissítés a biztonsági rés elhárítása érdekében javítja azt a módot, ahogyan a Windows Media Runtime inicializálja a funkciókat a tömörített hangfájlban.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**    
Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről.

**A frissítés telepítésének ellenőrzése**

-   **Microsoft Baseline Security Analyzer**

    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        
		**Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.  
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        
		**Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.

    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.
        
		**Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.  
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.
        
		**Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.

    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
