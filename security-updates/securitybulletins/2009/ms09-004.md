A Microsoft számú biztonsági közleménye MS09-004 - Fontos
=========================================================

A Microsoft SQL Server biztonsági rése távoli kódfuttatást tehet lehetővé (959420)
----------------------------------------------------------------------------------

Közzétéve: 2009. február 10. | Frissítve: 2009. március 18.

**Verzió:** 1.2

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Microsoft SQL Server közvetlenül jelentett biztonsági rését szünteti meg. A biztonsági rés akkor ad lehetőséget távoli kódfuttatásra, ha nem megbízható felhasználók hozzáférnek az érintett rendszerhez, illetve ha azon az SQL-beszúrási biztonsági rést kihasználó támadás történik. A SQL Server 7.0 Service Pack 4, SQL Server 2005 Service Pack 3 és az SQL Server 2008 rendszereket a probléma nem érinti.

A biztonsági frissítés az SQL Server 2000, SQL Server 2005 Service Pack 2, Microsoft SQL Server 2000 Desktop Engine (MSDE 2000), SQL Server 2005 Express Edition, Microsoft SQL Server 2000 Desktop Engine (WMSDE) és Windows Internal Database (WYukon) támogatott kiadásaira nézve fontos besorolású. További tudnivalókat a közlemény későbbi részében az **Érintett és nem érintett szoftverek** cím alatt talál.

A biztonsági frissítés a biztonsági rés megszüntetése végett hitelesíti a bővített tárolt eljárásnak küldött paramétereket. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

A biztonsági frissítés az először a [961040. számú Microsoft biztonsági tanácsadóban](http://technet.microsoft.com/security/advisory/961040) ismertetett biztonsági rést is megszünteti.

**Javaslat.** A Microsoft a biztonsági frissítés lehető leghamarabb történő telepítését javasolja ügyfeleinek.

**Ismert problémák** A [Microsoft Tudásbázis 959420. számú](http://support.microsoft.com/kb/959420) cikkében olvasható, hogy a felhasználók a biztonsági frissítés telepítésekor milyen ismert problémákkal találkozhatnak. A cikkben a problémákra javasolt megoldások is szerepelnek.

#### Érintett és nem érintett szoftverek

Az itt felsorolt szoftvereket a verziók és változatok érintettségének felderítése érdekében teszteltük. A többi verzió, illetve kiadás támogatási életciklusa végére ért vagy nem érintett. Az egyes szoftverek és azok verzióinak támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.

**Miért módosították a biztonsági közleményt 2009. március 5-én?**    
A Microsoft a módosított közleményben tájékoztat az észlelési logika módosulásáról a Microsoft SQL Server 2000 Desktop Engine (WMSDE) alkalmazás Windows Server 2003 Service Pack 1 és Windows Server 2003 Service Pack 2 rendszeren futó változatai (KB960082), továbbá a Windows Server 2003 x64 Edition és Windows Server 2003 x64 Edition Service Pack 2 rendszeren futó változatok (KB960082) frissítőcsomagjaiban. Az észlelési logika módosítása megszüntet egy telepítési problémát, amely hibát okozhatna a Microsoft SQL Server 2000 Desktop Engine (WMSDE) frissítésekor Active Directory áttelepítési eszközt (ADMT) vagy automatikus telepítési szolgáltatásokat (ADS) használó rendszereken. Ez csak észlelési módosítás, amely nincs hatással az eredeti közzététel fájljaira. A rendszerüket sikeresen frissített felhasználóknak nem kell újra telepíteniük a frissítést.
  
**Hol találhatók a fájlinformációs adatok?**    
A fájlinformációs adatok a [Microsoft Tudásbázis 959420. számú cikkében](http://support.microsoft.com/kb/959420) találhatók.
  
**Az SQL verziómhoz GDR és QFE frissítés is megtalálható. Hogyan válasszam ki a megfelelő frissítést?**    
Nézze meg az SQL Server verziószámát. Az SQL Server verziószám meghatározásának részleteit lásd a [Microsoft Tudásbázis 321185 sz.](http://support.microsoft.com/kb/321185) cikkében.
  
Ezután az alábbi táblázatban keresse ki azt a tartományt, amelybe az SQL Server verziószám tartozik. Az ennek megfelelő frissítést kell telepítenie.
  


**Megjegyzés** Ha az SQL Server verziószám nem esik bele az alábbi táblázat egyik tartományába se, az SQL Server verzió már nem támogatott. Frissítsen a legújabb Service Pack csomagra vagy SQL Server termékre, és vegye használatba a jelenlegi és a jövőbeni biztonsági frissítéseket.

**Felajánlják a frissítést az SQL Server fürtökhöz?**  
Igen. A frissítést az SQL Server 2000 és SQL Server 2005 fürtbe rendezett példányaihoz is felajánljuk. Az SQL Server fürtökhöz tartozó frissítés felhasználói közreműködést igényel.

Ha az SQL Server 2000 vagy az SQL Server 2005 fürtben passzív csomópont van, a Microsoft azt javasolja, hogy először az aktív csomópontot vizsgálja meg, és azon alkalmazza a frissítést, majd ezt követően a passzív csomóponton. Ha az összes összetevőt frissítette a passzív csomóponton, a rendszer nem ajánlja fel többet a frissítést.

**Az SQL Server 2000 és MSDE 2000 rendszerre vonatkozó hivatkozási táblázatok** **Tudnivalók a biztonsági frissítésről című része jelzi, hogy melyik frissítésnél van esetleg szükség újraindításra. Honnan tudhatom meg, hogy az adott frissítés igényli-e a rendszer újraindítását?**  
SQL Server 2000 és MSDE 2000 esetében függő szolgáltatás vehető igénybe. A függő szolgáltatás nem indul újra, noha a fölérendelt szolgáltatást leállították és újraindították. Vagyis a sikeres telepítést követően feltétlenül újra kell indítani a rendszert, hogy a függő szolgáltatás biztosan elinduljon. Az esetlegesen működő függő szolgáltatások azonosításának részleteit lásd a [Microsoft Tudásbázis 953741 sz.](http://support.microsoft.com/kb/953741) cikkében.

**A frissítések az SQL Server 2000, MSDE 2000 és SQL Server 2005 letiltott példányaira is érvényesek?**  
Nem. Csak azok az SQL Server 2000, MSDE 2000 és SQL Server 2005 példányokat frissíti a telepítő, melyek indítási típusa nem **letiltott**. A letiltott szolgáltatások részleteit lásd a [Microsoft Tudásbázis 953740. sz. cikkében](http://support.microsoft.com/kb/953740).

**A közleményben tárgyalt szoftvernél korábbi verzióval rendelkezem. Mi a teendő?**  
Az említett szoftververziók és -kiadások érintettségét teszteléssel állapították meg. A többi verzió illetve kiadás támogatási életciklusa végére ért. Az egyes szoftverek és azok verzióinak támogatási életciklusáról a [Microsoft termékek terméktámogatási életciklusát ismertető webhelyen](http://go.microsoft.com/fwlink/?linkid=21742) talál felvilágosítást.

A szoftver korábbi verziói és kiadásai felhasználóinak azt javasoljuk, hogy a potenciális biztonsági veszélyek elkerülése érdekében sürgősen térjenek át valamelyik támogatott verziójú illetve kiadású szoftverre. Az SQL Server sp\_replwritetovarbin eljárásának korlátozott memóriafelülírási biztonsági rése (CVE-2008-5416)
-------------------------------------------------------------------------------------------------------------

A paraméterek ellenőrzése az SQL Server alkalmazás "sp\_replwritetovarbin" bővített tárolt eljárásában távoli kódfuttatást okozó biztonsági rést nyit. A biztonsági rés akkor ad lehetőséget távoli kódfuttatásra, ha nem megbízható felhasználók hozzáférnek az érintett rendszerhez, illetve ha azon SQL-beszúrási biztonsági rés található. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását. Ezt követően programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de új fiókokat is létrehozhat.

A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2008-5416](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-5416).

#### Az SQL Server sp\_replwritetovarbin eljárásának korlátozott memóriafelülírási biztonsági résének (CVE-2008-5416) hibáját enyhítő tényezők

Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:

-   A probléma nem érinti a Microsoft SQL Server 7.0 Service Pack 4, Microsoft SQL Server 2005 Service Pack 3 és Microsoft SQL Server 2008 támogatott verzióit.
-   A biztonsági rés névtelen felhasználó által nem használható ki. A támadónak a biztonsági rés kihasználásához hitelesítést kell végrehajtania, vagy hitelesítésre képes webes alkalmazás SQL-beszúrási biztonsági rését kell kihasználnia.
-   Alapértelmezés szerint az MSDE 2000 és az SQL Server 2005 Express rendszer nem engedélyezi a távoli kapcsolatot. A biztonsági rés kihasználásához a hitelesített támadónak a támadás helyben kell kezdeményeznie.

#### Az SQL Server sp\_replwritetovarbin eljárásának korlátozott memóriafelülírási biztonsági résének (CVE-2008-5416) lehetséges megoldásai

A Microsoft alaposan megvizsgálta a következő lehetséges megoldásokat, és tájékoztatja a felhasználókat, amennyiben a megoldás csökkenti a működőképességet:

#### Gyakran feltett kérdések az SQL Server sp\_replwritetovarbin eljárásának korlátozott memóriafelülírási biztonsági rése (CVE-2008-5416) kapcsán

**Mire terjed ki a biztonsági rés hatása?**  
A biztonsági rés távolról történő kódfuttatást tehet lehetővé. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.

**Mi okozza a biztonsági rést?**  
Az "sp\_replwritetovarbin," MSSQL bővített tárolt eljárásban érvénytelen paraméterellenőrzés miatt hitelesített távoli végrehajtást okozó biztonsági rés található.

**Mire való az sp\_replwritetovarbin bővített tárolt eljárás?**  
Az sp\_replwritetovarbin bővített tárolt eljárás frissítendő előfizetők tranzakciós replikációjára szolgál, amennyiben az előfizetés @update\_mode = 'failover' illetve 'queued tran' körülmény mellett történt.

**Mire használhatja a biztonsági rést a támadó?**  
A biztonsági rést sikeresen kihasználó támadó mesterséges kódot futtathat az érintett rendszeren. Ezt követően a támadó programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de korlátozás nélküli jogosultságokkal rendelkező új fiókokat is létrehozhat.

**Hogyan használhatja ki a támadó a biztonsági rést?**  
A támadó az érintett bővített tárolt eljárást speciális módon elérő, érvénytelen paramétert tartalmazó kérést hozhat létre, mely a halom túlcsordulását és kódfuttatási lehetőséget okoz.

**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**  
A biztonsági rés a leginkább olyan rendszereken használható ki, ahogy a kiszolgálók engedélyezik a nem megbízható felhasználóknak a kérések betöltését és végrehajtását.

Az MSDE 2000 illetve SQL Server 2005 Express rendszer felhasználóit érinti a távoli támadás kockázata, amennyiben módosították az alapértelmezett telepítést a távkapcsolatok engedélyezése irányában, ha engedélyezték nem megbízható felhasználók hozzáférését az MSDE 2000 illetve SQL Server 2005 Express rendszerhez, illetve ha MSDE 2000 vagy SQL Server 2005 Express rendszerüket érinti a SQL-beszúrás biztonsági rése.

Az érintett Microsoft SQL Server szoftvert futtató rendszerek minden verzióját érinti a biztonsági rés kihasználásának kockázata, ha egy rosszindulatú felhasználónak sikerül bejelentkeznie. Ezen felül azok az SQL Server háttéradatbázisok is érintettek, amelyeknél megvan az SQL-beszúrás biztonsági rés kockázata. Az SQL-beszúrásról bővebben lásd a [Microsoft biztonsági tanácsadóját (954462)](http://technet.microsoft.com/security/advisory/954462).

**Mi a frissítés feladata?**  
A frissítés a biztonsági rés megszüntetése végett megfelelően hitelesíti a bővített tárolt eljárásnak küldött paramétereket.

**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**  
Igen. A biztonsági rés nyilvánosságra került, A normál biztonsági rés besorolást és a [CVE-2008-5416](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-5416) azonosítót kapta. A biztonsági rést elsőként a [Microsoft 961040. számú biztonsági tanácsadója](http://technet.microsoft.com/security/advisory/961040) ismertette.

**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**  
Nem. E közlemény eredeti verziójának kiadásáig a Microsoft találkozott ugyan olyan nyilvánosságra hozott leírásokkal, amelyek tartalmazták egy sikeres támadás forgatókönyvét, de nem kapott olyan információt, amely arra utalt volna, hogy ezt a biztonsági rést nyilvános támadásra használták.

Az Application Compatibility Toolkit (ACT) tartalmazza a Microsoft Windows Vista, a Windows Update, a Microsoft Security Update vagy a Windows Internet Explorer új verziójának központi telepítése előtt az alkalmazáskompatibilitási problémák kiértékeléséhez és enyhítéséhez szükséges eszközöket és dokumentációt.
