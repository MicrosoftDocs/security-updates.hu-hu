A Microsoft számú biztonsági közleménye MS12-019 - Közepes
==========================================================

A DirectWrite szolgáltatásmegtagadást okozó biztonsági rése (2665364)
---------------------------------------------------------------------

Közzétéve: 2012. március 13.

**Verzió:** 1.0

### Általános tudnivalók

#### Összefoglalás

A biztonsági frissítés a Windows DirectWrite nyilvánosan jelentett biztonsági rését szünteti meg. Azonnali üzenetkezelőre épülő támadás esetén a biztonsági rés szolgáltatásmegtagadást tehet lehetővé, ha a támadó közvetlenül egy azonnali üzenetkezelő ügyfélprogramnak speciálisan kialakított Unicode karaktersort küld. A megtámadott alkalmazás inaktív állapotba kerülhet, amikor a DirectWrite leképezi a speciálisan kialakított Unicode karaktersort.

Ez a biztonsági frissítés a Windows Vista, Windows Server 2008 (a Windows Server 2008 Itanium-alapú rendszerekhez kivételével), Windows 7 és Windows Server 2008 R2 összes támogatott kiadására nézve Mérsékelt besorolású. További információkért lásd az **Érintett és nem érintett szoftverek** című részt.

A biztonsági frissítés a biztonsági rés megszüntetéséhez módosítja az DirectWrite Unicode karaktereket érintő leképezési eljárását. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat:** A felhasználók nagy részénél be van kapcsolva az automatikus frissítés; ilyenkor nincs más teendő, mert a rendszer automatikusan letölti és telepíti a biztonsági frissítést. Ha nem használ automatikus frissítést, keresse meg a vonatkozó frissítést és telepítse manuálisan. Az automatikus frissítés további beállításairól lásd a [Microsoft Tudásbázis 294871. cikkét](http://support.microsoft.com/kb/294871).

Ha a rendszergazda, vállalati telepítő vagy végfelhasználó manuálisan szeretné telepíteni a biztonsági frissítést, a Microsoft az azonnali telepítést javasolja frissítéskezelő szoftver vagy a [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) szolgáltatás igénybevételével.

Lásd még a közlemény **Észlelési és telepítési eszközök, útmutatás** fejezetét.

A DirectWrite alkalmazás szolgáltatásmegtagadást okozó biztonsági rése - CVE-2012-0156  
--------------------------------------------------------------------------------------
  
A DirectWrite speciálisan kialakított Unicode karaktersorokat érintő leképezési módja szolgáltatásmegtagadást lehetővé tévő biztonsági rést okoz. A biztonsági rést sikeresen kihasználó támadó a kiszemelt alkalmazást a válaszadás megszakítására kényszerítheti.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2012-0156](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0156).
  
#### A DirectWrite alkalmazás szolgáltatásmegtagadást okozó biztonsági résének (CVE-2012-0156) súlyosságát enyhítő tényezők
  
A Microsoft egyelőre nem talált enyhítő tényezőket a biztonsági rés megszüntetésére.
  
#### Lehetséges megoldások a DirectWrite alkalmazás szolgáltatásmegtagadást okozó biztonsági résének (CVE-2012-0156) kiküszöbölésére
  
A Microsoft egyelőre nem talált lehetséges megoldást a biztonsági rés megszüntetésére.
  
#### Gyakori kérdések a DirectWrite alkalmazás szolgáltatásmegtagadást okozó biztonsági résével (CVE-2012-0156) kapcsolatban
  
**Mire terjed ki a biztonsági rés hatása?**  
Ez a biztonsági rés szolgáltatásmegtagadást okoz.
  
**Mi okozza a biztonsági rést?**  
A biztonsági rés akkor jön létre, amikor a DirectWrite hibásan képez le egy speciálisan kialakított Unicode karaktersort a memóriában.
  
**Mi az a DirectWrite?**  
A DirectWrite teljes Unicode, felbontásfüggetlen, többformátumú szövegleképezési API, amely támogatja a szövegek mérését, rajzolását és találatellenőrzését. A DirectWrite felületről bővebben [A DirectWrite bemutatása](http://msdn.microsoft.com/en-us/library/windows/desktop/dd371554(v=vs.85).aspx) című MSDN-cikkben olvashat.
  
**Mire használhatja a biztonsági rést a támadó?**  
A biztonsági rést sikeresen kihasználó támadó a kiszemelt alkalmazást (például Windows Live Messenger) a válaszadás megszakítására kényszerítheti.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**  
Webalapú támadás esetén a nem hitelesített támadónak olyan webhelyet kell üzemeltetnie, amely speciálisan kialakított Unicode karaktersort tartalmaz. Az érintett weboldalak és a felhasználó által biztosított tartalomnak, esetleg hirdetéseknek helyet adó weboldalak is tartalmazhatnak a biztonsági rés kihasználására használható különlegesen kialakított tartalmat. A biztonsági rés kihasználásához a különlegesen kialakított Unicode karaktersort olyan alkalmazásnak kell elemeznie, amely DirectWrite segítségével képezi le a webes tartalmakat. Az érintett alkalmazás inaktív állapotba kerülhet, amikor a DirectWrite leképezi a speciálisan kialakított Unicode karaktersort. A támadónak semmiképpen sincs lehetősége arra, hogy a felhasználót a különlegesen kialakított Unicode karaktersort tartalmazó tartalomnak otthont adó webhely megnyitására kényszerítse. Csupán megkísérelheti rávenni a felhasználókat arra, hogy kattintsanak az e-mailben vagy azonnali üzenetben megjelenő hivatkozásra, amelyen keresztül a támadó weboldalára jutnak.
  
Azonnali üzenetkezelőre épülő támadás esetén a nem hitelesített támadó úgy használhatja ki a biztonsági rést, hogy speciálisan kialakított Unicode karaktersort küld közvetlenül egy azonnali üzenetkezelő ügyfélprogramba, például a Windows Live Messenger alkalmazásba. A megtámadott alkalmazás inaktív állapotba kerülhet, amikor a DirectWrite leképezi a speciálisan kialakított Unicode karaktersort.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**  
A biztonsági rés elsősorban a munkaállomásokat és terminálkiszolgálókat veszélyezteti. A kiszolgáló nagyobb kockázatnak van kitéve, ha a rendszergazda engedélyezi a felhasználóknak a kiszolgálóra történő bejelentkezést és a programok futtatását. Ezt azonban a gyakorlati tanácsok értelmében feltétlenül kerülni kell.
  
**Mi a frissítés feladata?**  
A frissítés a biztonsági rés megszüntetéséhez módosítja az DirectWrite Unicode karaktereket érintő leképezési eljárását.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés az említett közlemény kiadásának idején?**  
Igen. A biztonsági rés nyilvánosságra került, A normál biztonsági rés besorolást és a [CVE-2012-0156](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0156) azonosítót kapta.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**  
Nem. A Microsoft nem kapott olyan információt, hogy a biztonsági közlemény eredeti változatának megjelenéséig a biztonsági rést felhasználták volna nyilvános támadásra

#### Microsoft Active Protections Program (MAPP)
  
Az ügyfelek védelmének növelése érdekében a biztonsági szoftverekkel foglalkozó nagyobb gyártók a Microsoft a havi biztonsági közlemények megjelenése előtt ellátja biztonsági résekkel kapcsolatos információkkal. Így a biztonságiszoftver-gyártók a kapott biztonsági résekkel kapcsolatos információt felhasználhatják ügyfeleik hatásosabb védelmére az olyan biztonsági szoftverek vagy eszközök (például víruskeresők, hálózati behatolásérzékelő rendszerek vagy kiszolgálóalapú behatolásvédelmi rendszerek) frissítése által. A biztonságiszoftver-gyártók által kiadott aktív védelem elérhetőségéről a [Microsoft Active Protections Program (MAPP) Partners](http://go.microsoft.com/fwlink/?linkid=215201) webhelyén található listán szereplő és a programban részt vevő gyártók aktív védelmi webhelyein tájékozódhat.
