A Microsoft számú biztonsági közleménye MS08-062 - Fontos
=========================================================

A Windows Internet Printing Service biztonsági rése, amely távoli programkódfuttatást tesz lehetővé (953155)
------------------------------------------------------------------------------------------------------------

Közzétéve: 2008. október 14. | Frissítve: 2008. október 29.

**Verzió:** 2.2

### Általános tudnivalók

#### Összefoglalás

A frissítés a Windows Internet Printing Service egy közvetlenül jelentett biztonsági rését szünteti meg, amely távoli kódfuttatásra adott lehetőséget. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását. Ezt követően programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de új fiókokat is létrehozhat.

Ez a frissítés a Windows 2000, a Windows XP, a Windows Server 2003 és a Windows Server 2008 rendszerek összes támogatott kiadására nézve fontos besorolásúnak minősül. További tudnivalókért tekintse meg az **Érintett és nem érintett szoftverek** c. részt.

A biztonsági frissítés úgy szünteti meg a biztonsági rést, hogy megváltoztatja az Internet Printing Protocol (IPP) szolgáltatásban a memórialefoglalás módját. A biztonsági réssel kapcsolatban bővebben az adott problémával foglakozó gyakran feltett kérdéseket ismertető szakaszban, **A biztonsági réssel kapcsolatos tudnivalók** című részben olvashat.

**Javaslat.** A Microsoft a biztonsági frissítés lehető leghamarabb történő telepítését javasolja ügyfeleinek.

**Ismert problémák** Nincsenek

**Miért módosították a közleményt 2008. október 29-én?**  
A Microsoft azért javította a közleményt, hogy tájékoztassa a felhasználókat: a Windows Server 2008 Itanium alapú rendszerekhez, illetve a Windows Vista összes támogatott kiadásának frissítőcsomagjai elérhetők a Windows Update, a Microsoft Update, a Windows Software Update Services (WSUS), a Systems Management Server és a System Center Configuration Manager segítségével.. A közlemény első közzétételekor az ezekhez a rendszerekhez tartozó frissítések csak a Microsoft Letöltőközpontból voltak elérhetők.

**Miért módosították a közleményt 2008. október 16-án?**  
A Microsoft módosította a közleményt, melyben tisztázza, hogy a Windows Internet Printing Service a rendszerszintű jogosultságokkal működő nyomtatásisor-kezelő szolgáltatás alatt fut. A jelen biztonsági közleményben tárgyalt biztonsági rést sikeresen kihasználó támadó rendszerjogosultsággal futtathat mesterséges kódot, és átveheti az érintett rendszer teljes ellenőrzését.

**Miért módosították a közleményt 2008. október 15-én?**  
A Microsoft törölte a közleményből a Windows Server 2008 Itanium alapú rendszerekhez súlyossági besorolását. A Microsoft ezenfelül a frissített közleményben tisztázza, hogy a Windows Server 2008 Itanium alapú rendszerekhez frissítése a Microsoft Letöltőközpontból érhető el, és amint lehetséges, a többi terjesztési csatornán keresztül is közzéteszi.

**Miért szerepel a Windows Server 2008 Itanium alapú rendszerekhez és a Windows Vista rendszer az érintett szoftvereket tartalmazó táblázatban, ha nincs súlyossági besorolása?**  
A Windows Server 2008 Itanium alapú rendszerekhez és a Windows Vista összes támogatott kiadása tartalmazza a megosztott nyomtatásisor-kezelő összetevőt, amelynek frissítését ez a biztonsági frissítés tartalmazza. Mivel azonban a Windows Server 2008 Itanium alapú rendszerekhez és a Windows Vista nem támogatja az internetes nyomtatást, a biztonsági közleményben tárgyalt támadási pont nem érintett. A Windows Server 2008 Itanium alapú rendszerekhez és a Windows Vista felhasználói számára a jövőben felfedezett esetleges új támadási pontok elleni mélyreható védekezési lépésként javasolt a biztonsági frissítés alkalmazása.

Az IPP szolgáltatás egészszám-túlcsordulási biztonsági rése - CVE-2008-1446  
---------------------------------------------------------------------------
  
A távoli kódfuttatást lehetővé tévő biztonsági rés azokban a Windows-rendszerekben van jelen, melyeken Microsoft Internet Information Services (IIS) fut, és engedélyezték a Windows Internet Printing Service használatát. Ez hiba lehetővé teszi, hogy távoli hitelesített felhasználó tetszőleges kódot futasson az érintett rendszeren.
  
A biztonsági rés standard bejegyzésének megtekintéséhez a normál biztonsági rés besorolási listáján lásd: [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).
  
#### Az IPP szolgáltatás egészszám-túlcsordulási biztonsági rése (CVE-2008-1446) súlyosságát enyhítő tényezők
  
Az enyhítő tényezők közé sorolható egy beállítás, konfiguráció vagy általánosan bevált gyakorlat, amely alapértelmezett állapotában csökkentheti a biztonsági rés kihasználásának valószínűségét. A következő enyhítő tényezők hasznosnak bizonyulhatnak:
  
-   A Microsoft Windows 2000 és a Windows XP IIS szolgáltatásában az Internet Printing Protocol (IPP) alapértelmezetten engedélyezett. Az alapértelmezett fájlengedélyek azonban minden platformon úgy vannak beállítva, hogy az IPP csak a "Hitelesített" csoportban lévő felhasználóknak legyen elérhető. Emiatt a korlátozás miatt a nem hitelesített felhasználók alapértelmezés szerint nem rendelkeznek a biztonsági rés távoli kihasználásához szükséges mappaengedélyekkel.  
-   Az IIS Windows Server 2003 rendszerre telepített példányai az alapértelmezett konfigurációs beállításaikkal nem veszélyeztetettek. Csak azok a Windows Server 2003 telepítések veszélyeztettek, amelyek az IIS szolgáltatásban engedélyezték az Internet Printing Protocol ISAPI szűrőjét. A web alapú nyomtatókezeléshez is és az internetes nyomtatáshoz is szükség van az Internet Printing és az Active Server Pages összetevőre. Ha az Active Server Pages összetevő beállítása "Engedélyezett", az Internet Printing összetevőé pedig "Tiltott művelet", csak a web alapú nyomtatókezelés engedélyezett. A felhasználók nem csatlakozhatnak az internetes nyomtatás egyik megosztott nyomtatójához sem.  
-   A Windows Server Server 2008 rendszerű kiszolgálóra telepített IIS alapértelmezetten nem veszélyeztetett. A Windows Server 2008 rendszerrel működő számítógép lehet internetes nyomtatást használó ügyfélgép. Azonban a nyomtatási kérelmet feladó számítógép felhasználóit a nyomtatási kiszolgálónak hitelesítenie kell, mielőtt használhatnák a kiszolgálóhoz csatlakoztatott bármelyik nyomtatót. Ahhoz, hogy a Windows Server 2008 rendszerrel működő számítógép az internetes nyomtatást támogató kiszolgálóként szerepeljen, szükséges a Web Server (IIS) szerepkör, a Nyomtatási szolgáltatások és a nyomtatási szolgáltatásokon belül az internetes nyomtatási szerepkört is telepíteni.  
-   A biztonsági rés kihasználásához az is kell, hogy az IIS-kiszolgáló a kártékony számítógéppel az adatátvitelt a 445-ös porton bonyolítsa. Azok a rendszergazdák, akik letiltják az IIS kiszolgálójukról az internetre kimenő SMB-csatlakozásaikat, elkerülhetik, hogy a nem megbízható rendszerekről kihasználhassák ezt a biztonsági rést.
  
#### Az IPP szolgáltatás egészszám-túlcsordulási biztonsági rése (CVE-2008-1446) miatti kockázatok elkerülése
  
A lehetséges megoldás olyan beállítást vagy konfigurációs módosítást takar, amely magát a biztonsági rést nem szünteti meg, azonban a frissítés telepítése előtt segít az ismert támadási lehetőségek kivédésében. A Microsoft alaposan megvizsgálta a következő lehetséges megoldásokat, és tájékoztatja a felhasználókat, amennyiben a megoldás csökkenti a működőképességet:
  
-   **Az IPP szolgáltatás kikapcsolása**
  
    Az IPP szolgáltatás kikapcsolása az IIS 5.0 és az IIS 5.1 verzióban
  
    1.  Nyissa meg az **Internetes szolgáltatások kezelője** segédprogramot.  
    2.  A jobb egérgombbal kattintson az **Alapértelmezett webhely** feliratra, és válassza ki a **Tulajdonságok** menüelemet.  
    3.  Váltson a **Kezdőkönyvtár** lapra, és válassza ki a **Konfigurálás** elemet.  
    4.  Görgessen lefelé, és keresse meg a **.printer** kiterjesztést és az **msw3prt.dll** elérési útját.  
    5.  Kattintson az **Eltávolítás** gombra.  
    6.  Kattintson az **OK** gombra.  
    7.  Indítsa újra az IIS szolgáltatást.
    
    **Megjegyzés** Az Internetes nyomtatás csoportházirend és az Internetes szolgáltatások kezelője segítségével is konfigurálható. Ha ütközés van a csoportházirend-beállítások és az Internetes szolgáltatások kezelője beállításai között, a csoportházirend-beállítások élveznek elsőbbséget. Ha az Internetes szolgáltatások kezelője segítségével eltávolítja az Internetes nyomtatás lehetőséget, feltétlenül ellenőrizze, hogy a helyi vagy tartományszintű csoportházirendek nem engedélyezik-e újra. #### Gyakran feltett kérdések az IPP szolgáltatás egészszám-túlcsordulási biztonsági réséről (CVE-2008-1446)
  
**Mire terjed ki a biztonsági rés hatása?**  
A biztonsági rés távolról történő kódfuttatást tehet lehetővé. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását. Ezt követően programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de új fiókokat is létrehozhat.
  
**Mi okozza a biztonsági rést?**  
Az Internet Printing Protocol (IPP) funkció megvalósítása az IIS szolgáltatás ISAPI bővítményeként történt. Az IIS szolgáltatást futtató Windows-kiszolgálókon az IPP Microsoft rendszerbeli megvalósítása távolból elérhető hitelesítési biztonsági rést tartalmaz. Az IPP implementációba olyan egészszám-túlcsordulás keletkezhet, ami lehetővé teheti, hogy egy hitelesített támadó távoli kódfuttatást hajtson végre az érintett IIS kiszolgálókon.
  
**Mi az ISAPI?**  
Az ISAPI (Internet Services Application Programming Interface) olyan műszaki megoldás, amelyik lehetővé teszi, hogy a webfejlesztők bővítsék a webkiszolgálóik funkcióit olyan egyéni programkódok írásával, amelyek új szolgáltatásokat biztosítanak a webkiszolgálón. Az egyéni programkódokat lehet vagy az ISAPI szűrőbe implementálni, ha az új funkció alacsony szintű szolgáltatást nyújt, vagy az ISAPI bővítményeként, ha az új funkció magas szintű szolgáltatást nyújt. Ebben az esetben az érintett kód az ISAPI bővítménye.
  
**Mi itt a probléma az ISAPI bővítményével?**  
Az érintett ISAPI bővítmény is implementálja az Internet Printing Protocol (IPP) protokollt, ami RFC 2910 és 2911 által meghatározott ipari szabvány. Az IPP protokoll lehetőséget ad az internet HTTP protokollján keresztül nyomtatási szolgáltatások kérésére és a nyomtatási állapotok megismerésére. Az IPP használatával például az úton lévő felhasználó az interneten keresztül nyomtatási feladatot küldhet a vállalata hálózatán lévő nyomtatónak. A felhasználó azt is megnézheti, hogy a nyomtatási kérelme teljesítve lett-e hiba nélkül.
  
Az internetes nyomtatás beépített támogatását a Microsoft Windows 2000 vezette be. A Microsoft Windows 2000 implementációja lehetővé teszi, hogy a felhasználók közvetlenül URL-címre nyomtassanak, és a böngészőjükkel megtekintsék a nyomtatási feladat állapotadatait. Az internetes nyomtatás a Microsoft Windows 2000 Server verzióiban alapértelmezetten engedélyezett. Kiegészítő összetevőként telepíthető a Microsoft Windows 2000 Professional, a Windows XP, a Windows Server 2003 és a Windows Server 2008 rendszeren is.
  
**Mire használhatja a biztonsági rést a támadó?**  
A biztonsági rést sikeresen kihasználó támadó rendszerjogosultsággal mesterséges kódot futtathat a felhasználó rendszerén. Ha egy támadó kihasználja a biztonsági rést, teljes mértékben átveheti a rendszer irányítását. Ezt követően programokat telepíthet, illetve adatokat tekinthet meg, módosíthat és törölhet, de új fiókokat is létrehozhat.
  
**Hogyan használhatja ki a támadó a biztonsági rést?**  
A támadó küldhet olyan speciálisan kialakított HTTP POST kérelmet a veszélyeztetett webkiszolgálónak, amelyik eredményeként a kiszolgáló csatlakozik a támadó által vezérelt, IPP protokollt használó nyomtatónak látszó számítógéphez. A támadó ekkor olyan speciálisan kialakított válaszokat produkál, amelyekkel távoli kódfuttatást hajt végre a megtámadott kiszolgálón.
  
**Mely rendszerekre jelenti a legnagyobb veszélyt a biztonsági rés?**  
Ez a biztonsági rés elsősorban az IIS szolgáltatást és az engedélyezett Internet Printing Protocol (IPP) protokollt használó munkaállomásokat és kiszolgálókat veszélyezteti.
  
**Mi a frissítés feladata?**  
Ez a biztonsági frissítés úgy szünteti meg a biztonsági rést, hogy ellenőrzi a távoli számítógépekről kapott adatokat az Internet Printing Protocol (IPP) szolgáltatásban.
  
**Ismert volt-e a nyilvánosság előtt ez a biztonsági rés a biztonsági közlemény kiadásának idején?**  
Nem. A Microsoft ezt a célt szolgáló tájékoztatáson keresztül szerzett tudomást a biztonsági résről. A Microsoft nem kapott olyan információt, hogy e közlemény eredeti változatának megjelenéséig a biztonsági rést a nyilvánosság tudomására hozták volna.
  
**A közlemény kibocsátásáig kapott-e a Microsoft a biztonsági rés kihasználásáról tájékoztató értesítést?**  
Igen, a Microsoft figyeli az esetleges biztonsági rés kihasználására törekvő támadásokat. Meg kell azonban említeni, hogy a Microsoft nem látott nyilvánosságra hozott bizonyítékot sikeres támadási forgatókönyvekre e közlemény eredeti verziójának kiadásáig.

**A frissítés telepítésének ellenőrzése**

-   **Microsoft Baseline Security Analyzer**

    A Microsoft Baseline Security Analyzer (MBSA) eszközzel ellenőrizheti, hogy a biztonsági frissítés telepítve van-e az érintett számítógépen. További tájékoztatásért tekintse meg a közlemény korábbi részének **Észlelési és telepítési eszközök, útmutatás** című fejezetét.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.

        **Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.

        **Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Nem biztos, hogy ezek a rendszerleíró kulcsok az összes telepített fájlt tartalmazzák. Ne feledje azt sem, hogy ha a rendszergazda vagy az OEM gyártó beépíti a biztonsági frissítést a Windows rendszer telepítési forrásfájljaiba, a program esetleg hibásan hozza létre a rendszerleíró kulcsokat.

-   **Fájlverziószám ellenőrzése**

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.

        **Megjegyzés** Az operációs rendszer vagy a programok verziószámától függően elképzelhető, hogy a fájlinformációs táblázatban szereplő fájlok némelyike nem található meg a rendszerben.
    5.  A **Verzió** lapon hasonlítsa össze a rendszeren lévő fájl verziószámát a megfelelő táblázatban szereplő fájl verziószámával.

        **Megjegyzés** A telepítés során a fájl verziószámán kívül más attribútumok is megváltozhatnak. Ezen attribútumok fájlinformációs táblázatban szereplő adatokkal való összevetése a frissítés alkalmazásának nem ajánlott ellenőrzési módja. Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.

    Mivel a Microsoft Windows operációs rendszernek több verziója és kiadása is van, a lépések az egyes rendszereken eltérőek lehetnek. Ilyen esetben a megfelelő lépések végrehajtásához az adott termék dokumentációja ad segítséget.

    1.  Kattintson a **Start** menü **Keresés** parancsára.
    2.  A **Keresés eredménye** ablak **Keresés** ablaktáblájában válassza a **Minden mappa és fájl** elemet.
    3.  **A fájlnév része vagy egésze** mezőbe írja be a táblázatban szereplő megfelelő fájl nevét, majd kattintson a **Keresés** gombra.
    4.  A fájlok listájában kattintson jobb gombbal a megfelelő fájl nevére, majd kattintson a **Tulajdonságok** parancsra.

Előfordulhat az is, hogy a telepítés során bizonyos esetekben a rendszer fájlokat nevez át. Ha a fájl- vagy verzióadatok nem találhatók, ellenőrizze a frissítés telepített példányát egyéb módszerek segítségével.

-   **Rendszerleíró kulcsok ellenőrzése**

    A biztonsági frissítés által telepített fájlokat a **Hivatkozási táblázatban** feltüntetett rendszerleíróadatbázis-kulcsok segítségével is ellenőrizheti.
